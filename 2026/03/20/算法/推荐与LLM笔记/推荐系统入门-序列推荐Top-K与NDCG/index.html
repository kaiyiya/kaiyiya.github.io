推荐系统入门（一）：序列推荐、Top-K 与 NDCG@k

飞向月球的猫2026-03-202026-04-13

如果你熟悉 CV 里的分类 / 检测 / 分割，可以把推荐里最常见的一类问题概括成两件事：

  • 序列推荐：给定用户过去点击、购买、观看的物品序列，预测下一个最可能交互的物品（本质是「序列上的下一步预测」）。
  • Top-K 推荐：给定用户与上下文，从海量物品里排出前 K 个最可能被用户喜欢的条目（本质是排序）。

1. 核心任务：用例子建立直觉

1.1 序列推荐

设定：用户按时间顺序交互了物品 {牛奶, 面包, 麦片}，模型问：下一个最可能是？

  • 输入：历史序列（常编成 ID 序列，如 [12, 5, 88]）。
  • 输出：下一个物品的分布（对所有候选物品一个概率或分数），或直接取 argmax。

和 CV 的类比：有点像「给定视频前几帧，预测下一帧的语义标签」，只是这里的「帧」换成了离散物品 ID，目标常是分类式的 softmax over 物品表。

1.2 Top-K 推荐

设定：候选集里有 100 万个商品，模型要给当前用户排出「最有可能点开的 10 个」。

  • 输出不是单个类，而是有序列表（长度 K），顺序影响评价指标（见下文 NDCG）。

和 CV 的类比：接近图像检索或多标签排序——你关心的是「相关项是否排在前面」，而不是只猜一个 top-1 类。

2. 关键场景：I.I.D 暖启动 vs O.O.D 冷启动

场景 含义（直观） 例子
暖启动 / I.I.D 假设 训练、测试时用户和物品都「见过」，分布差不多 训练集里出现过用户 U 和物品 I，测试也让 U 给 I 打分
冷启动 / O.O.D 测试时出现新用户或新物品（或新域），和训练分布不一致 新上架商品从未被点击；新用户没有历史行为
跨域 / 多域推荐 利用源域（如图书）的共性，帮到目标域（如电影） 论文里「基酒 LoRA」一类思路：先在多域上学通用低秩适配，再接到新域

核心痛点（论文常见动机）：数据稀疏（交互少）、冷启动（新用户/新物品）、域迁移难（换一个业务场景模型就废）。

3. 评价指标：NDCG@k 与 HR@k（带小算例）

3.1 为什么需要 NDCG？

Top-K 是有序列表，只算「命中没命中」不够：相关物品排在第 1 位和第 10 位，用户体验不同。NDCG（归一化折损累积增益）同时奖励：

  • 排得靠前的命中；
  • 多个相关物品都被排到前面。

论文里常报 NDCG@1、NDCG@3：只考察推荐列表前 1 / 前 3 个位置的质量，适合移动端首屏、短列表场景。

3.2 DCG 与 NDCG（极简手工例）

假设真实与用户相关的物品集合为 {A, B}。模型给出的 Top-3 排序为：[A, X, B]。

对位置 $i=1,2,3$ 给予折损 $\log_2(i+1)$，相关位置增益为 1，不相关为 0：

$$
\mathrm{DCG} = \sum_{i=1}^{3} \frac{\mathrm{rel}_i}{\log_2(i+1)}
$$

此处 $\mathrm{rel}_1=1$（A 相关），$\mathrm{rel}_2=0$，$\mathrm{rel}_3=1$（B 相关）：

$$
\mathrm{DCG} = \frac{1}{\log_2 2} + 0 + \frac{1}{\log_2 4} = 1 + 0 + 0.5 = 1.5
$$

理想排序应为 [A, B, …]，同样取 Top-3 算 IDCG（把相关项尽量放最前得到的 DCG 上限），则：

$$
\mathrm{NDCG@3} = \frac{\mathrm{DCG}}{\mathrm{IDCG}}
$$

NDCG 取值在 $[0,1]$，越大越好。这样「A 在第 1 位」比「A 在第 3 位」得分更高，符合「排序质量」的直觉。

3.3 HR@k（Hit Rate）

HR@k：在前 k 个推荐里是否至少出现一个真实交互过的物品（0/1 或批量平均）。实现简单，但不区分「第 1 名命中」和「第 k 名命中」，常作辅助指标。

4. 经典范式与模型：抓到主线即可

4.1 协同过滤（CF）

思想：「兴趣相似的用户喜欢相似物品」或「相似物品会被同一批人喜欢」。基于用户–物品交互矩阵（点击、评分），用矩阵分解、近邻、神经网络等估计缺失项。

微型例子：用户 U1、U2 都买过 {键盘, 鼠标}，U1 还买了鼠标垫，则给 U2 推鼠标垫——这就是物品共现 + 用户相似的朴素逻辑。

4.2 序列模型：GRU4Rec / SASRec

  • GRU4Rec：用 RNN（GRU）读入历史点击序列，下一步预测物品。
  • SASRec：用 Self-Attention 建序列，捕捉长依赖，仍是「序列 → 下一项 / 多项打分」。

读论文时只需记住：输入是行为序列，核心是建模时序依赖；细节实现可以后补。

5. 与 CV 的衔接（迁移学习视角）

CV 里熟悉的 推荐里对应
多类分类 下一物品预测（超大类表）
图像检索 / mAP Top-K 排序、NDCG@k
域自适应、长尾 冷启动、稀疏、跨域推荐

一句话：推荐的「Top-K 排序」与检索里的「相关样本是否排在前面」是同一类排序评价思维；NDCG 与 MAP/Recall@k 都在衡量排序列表与真实偏好的吻合程度，只是定义细节不同。

6. 本篇小结

  • 序列推荐 = 历史交互序列 → 预测下一步（或打分）。
  • Top-K = 全库排序取前 K，必须用排序指标（如 NDCG@k）评价。
  • 暖 vs 冷 = 分布是否偏移、新用户/新物品是否出现。
  • CF + 序列模型（GRU4Rec/SASRec） = 工业与论文里最常出现的基线思路。
