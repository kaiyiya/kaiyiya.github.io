一、CSS3过渡和动画

1. 过渡
1）过渡和动画介绍

*

  • 功能区别：过渡和动画都能让CSS属性变化更平滑，但过渡更简单，能用过渡尽量用过渡
  • 适用场景：过渡适合简单的属性变化，动画适合复杂的多阶段变化
2）过渡示例

*

  • 基本实现：
    • 初始状态：设置宽高100px，背景色#f40
    • 悬停状态：宽高200px，背景色#008c8c
    • 添加transition: 500ms使变化平滑
  • 注意事项：
    • 过渡属性应写在常规样式中而非:hover，否则移出时会失去过渡效果
    • 时间单位可用s(秒)或ms(毫秒)，如0.5s和500ms等效
3）过渡的用法

*

  • 完整语法：transition: 过渡属性 持续时间 过渡函数 过渡延迟
  • 过渡属性：
    • 指定属性名(如transform)则仅对该属性生效
    • 使用all或不指定则对所有属性生效
  • 持续时间：
    • 必须带单位：3s表示3秒，500ms表示500毫秒
    • 建议值：0.3s-1s之间效果最佳
4）过渡函数

*

  • 预设值：
    • ease(默认)：先加速后减速
    • ease-in：缓慢开始，逐渐加速
    • ease-out：快速开始，缓慢结束
    • ease-in-out：缓慢开始和结束
    • linear：匀速变化
  • 自定义：可通过cubic-bezier()自定义贝塞尔曲线
  • 调试技巧：在开发者工具中实时调整曲线预览效果
5）过渡延迟

*

  • 语法：在持续时间后添加延迟时间，如transition: 500ms 1s
  • 效果：触发变化后等待指定时间才开始过渡
  • 应用场景：创建分阶段动画效果，避免多个元素同时变化
6）事件监听
  • JS事件：
    • transitionstart：过渡开始时触发
    • transitionend：过渡结束时触发
  • 应用：可在这些事件回调中执行其他操作，实现复杂交互效果

3）动画的细节

  • 动画的基本用法

    • 规则定义：使用@keyframes定义动画规则，0%可简写为from，100%可简写为to
    • 应用语法：animation: 规则名 持续时间是最简写法，例如animation: test 2s
    • 扩展参数：完整语法为animation: 规则名 持续时间 重复次数 时间函数 动画方向 延迟时间

  • 重复次数与播放效果

    *

    • 次数设置：通过数字指定播放次数，如animation: test 2s 2表示播放两次
    • 播放机制：每次播放都从0%到100%完整执行，连续播放时会有”跳回”效果
    • 功能对比：比transition功能更强大但代码量更多，优先使用过渡效果

  • 无限次播放与过渡对比

    • 无限循环：设置infinite参数使动画无限播放，如animation: test 2s infinite
    • 使用建议：简单效果用transition，复杂动画才使用animation
    • 性能注意：无限循环动画会持续消耗系统资源

  • 时间函数与调整

    *

    • 常用函数：支持ease、ease-in、ease-out、ease-in-out等预设值
    • 自定义调节：可通过cubic-bezier()函数自定义时间曲线，如cubic-bezier(0.42,0,0,1.38)
    • 调试方法：在开发者工具的Styles面板中实时调整参数观察效果

  • 动画方向与交替反向

    *

    • 方向控制：使用alternate参数实现交替反向播放
    • 播放模式：第一次正向(0%→100%)，第二次反向(100%→0%)，第三次再正向
    • 视觉表现：消除动画重复时的”跳跃感”，使过渡更加平滑自然

  • 延迟播放的设置

    *

    • 延迟语法：在参数列表最后添加时间值，如1s表示延迟1秒
    • 执行特点：只在动画首次播放时延迟，后续循环不受影响
    • 应用场景：用于控制多个动画的启动时机，创造错落有致的视觉效果
4）应用案例

  • 例题：鼠标移至图片渐变示例

    • CSS3过渡效果实现
      *
      • 基础样式设置：
        • 使用border-radius: 50%将图片变为圆形（50%与100%效果相同）
        • 初始设置opacity: 0.5实现半透明效果
        • 通过margin: 50px auto实现居中显示
      • hover状态变化：
        • box-shadow: 0 0 3px rgb(0,0,0)添加阴影效果
        • opacity: 1取消透明度
        • transform: scale(1.5) rotate(1turn)实现放大1.5倍并旋转360度
    • 过渡属性详解
      *
      • transition使用要点：
        • 必须设置在初始状态（非hover状态）
        • 简写属性：transition: 0.5s表示所有属性过渡0.5秒
        • 支持属性：仅限数值类属性（颜色、尺寸、透明度等）
      • 变形组合技巧：
        • 多种变形用空格分隔：transform: scale() rotate()
        • 执行顺序影响最终效果（先旋转后平移与先平移后旋转不同）
    • 注意事项
      *
      • 不支持的属性：
        • 非数值属性如cursor、display、position等
        • 枚举值属性无法实现渐变效果
      • 开发建议：
        • 新手应养成查阅文档的习惯（不丢人且必要）
        • 使用Live Server测试绝对路径资源
        • 实际开发中建议明确指定过渡属性而非使用all

  • 例题：点击屏幕圆点移动示例

    • 基础实现
      *

      • 定位方式：使用position: fixed固定定位使元素脱离文档流

      *
      • 过渡属性：添加transition: 0.5s使位置变化产生平滑动画效果
      • 中心点修正：通过减去元素宽高的一半（15px）使鼠标点击位置对应元素中心而非左上角
      • 性能优化：建议使用transform: translate()代替直接修改left/top属性，避免触发重排

    • 定位方式选择：固定定位适合需要脱离文档流的交互元素
    • 坐标系统：clientX/Y获取的是相对于视口的坐标值
    • 过渡效果：CSS过渡可以自动处理属性变化的中间状态

  • 例题：手风琴菜单示例

    • 实现原理
      *
      • 核心思路：通过控制子菜单高度实现展开/收起效果
        • 初始状态：所有子菜单高度设为0，并设置overflow:hidden
        • 点击触发：将当前子菜单高度设为固定值150px
        • 过渡效果：给子菜单添加transition属性实现平滑动画
      • 关键限制：
        • 必须使用固定高度值（如150px），不能使用auto
        • 使用auto高度将无法产生过渡效果，需要特殊技巧解决
    • CSS实现
      *
      • 基础样式：
        • 容器圆角：.menu-container {border-radius: 5px;}
        • 子菜单初始状态：.sub-menu {height:0; overflow:hidden;}
        • 过渡效果：.sub-menu {transition: 0.3s;}
      • 注意事项：
        • 必须同时设置overflow:hidden，否则内容会溢出显示
        • 过渡时间0.3秒是常用值，可根据需求调整
    • JavaScript交互实现
      • 事件绑定：
        • 精准定位：给h2元素而非整个menu绑定点击事件
        • 元素查找：document.getElementsByClassName(‘menu’)获取所有菜单
        • 循环处理：遍历每个menu元素，找到其中的h2元素
      • 点击处理逻辑：
        • 获取目标：this.nextElementSibling获取相邻的ul元素
        • 高度切换：通过判断clientHeight决定设置150px或0
        • 其他菜单关闭：循环所有子菜单，排除当前ul后设置height=0
    • 常见问题解决
      • 事件未触发：
        • 检查元素选择是否正确，特别是h2元素的获取
        • 使用console.log输出验证元素查找结果
      • 过渡效果缺失：
        • 确认transition属性已正确添加到.sub-menu
        • 检查是否意外覆盖了transition属性
      • 内容显示异常：
        • 确保同时设置了height:0和overflow:hidden
        • 检查是否有其他CSS规则影响了布局
    • 扩展优化
      • 性能优化：
        • 将sub-menu查询结果缓存，避免重复DOM查询
        • 使用事件委托减少事件监听器数量
      • 用户体验：
        • 添加箭头图标指示展开状态
        • 考虑添加点击区域扩大处理
        • 可增加active类名辅助样式控制
      • 兼容性处理：
        • 考虑使用classList代替直接style修改
        • 为不支持transition的浏览器提供降级方案

二、CSS3过渡和动画

1. 例题：手风琴效果

*

  • 实现原理：通过JS控制子菜单高度变化，配合CSS过渡效果实现展开/收起动画
  • 关键代码：
    • 固定高度150px（auto无法产生过渡效果）
    • transition: 0.3s设置过渡时间
    • JS通过clientHeight判断当前状态
    • height在0和150px间切换实现动画
  • 注意事项：必须设置固定高度值，auto高度无法产生过渡效果
2. 例题：球动画示例

*

  • 动画定义：
    • 使用@keyframes定义名为jump的动画
    • 0%状态：transform: scale(1.3, 0.7)（横向拉伸纵向压缩）
    • 100%状态：transform: translateY(-100px) scale(0.7, 1.3)（上移并反向拉伸）
  • 动画应用：
    • animation: jump 0.5s infinite alternate
    • infinite表示无限循环
    • alternate表示交替反向播放
  • 特殊语法：
    • 0%可用from代替
    • 100%可用to代替
  • 设计技巧：通过组合scale和translate实现弹跳效果
3. 例题：加载中动画示例
  • 实现步骤：
    • 为每个加载条设置border-radius圆角
    • 定义@keyframes loading动画：
      • 0%：transform: scaley(1)
      • 100%：transform: scaley(1.5)
    • 应用动画：animation: loading 0.5s infinite alternate
  • 错开效果：
    • 通过animation-delay为每个元素设置不同延迟
    • 例如：.dot2 {animation-delay: 0.2s}
  • 复合属性：animation是包含duration、timing-function、delay等的简写属性
4. 例题：荡秋千动画

*

  • 关键实现：
    • 定义@keyframes swing：
      • from {transform: rotate(45deg)}
      • to {transform: rotate(-45deg)}
    • 设置transform-origin: center top改变旋转支点
  • 动画参数：
    • animation: swing 1s infinite alternate ease-in-out
    • 使用ease-in-out使运动更自然
  • 视觉技巧：通过改变旋转角度和支点位置模拟真实物理摆动

三、知识小结

CSS动画(keyframes) 预定义CSS变化规则，可控制播放次数和方向 @keyframes jump { 0%{…} 100%{…}} animation: jump 1s infinite alternate; 复杂动画效果、循环动画 ⭐⭐⭐
变形属性(transform) 实现缩放、旋转、平移等视觉效果 transform: scale(1.5) rotate(45deg); transform-origin: center top; 元素变形、动画基础 ⭐⭐
过渡与动画对比 过渡简单但功能有限，动画复杂但功能强大 transition适合简单交互，animation适合复杂序列 根据需求选择技术方案 ⭐⭐

实践案例4-秋千动画 通过改变变形原点实现摆动效果 transform-origin: center top; @keyframes swing { from{transform: rotate(45deg);} to{transform: rotate(-45deg);}} 装饰性动画、界面元素 ⭐⭐
