2）根据元素名称获取DOM ﻿08:27
  • document.getElementsByTagName()
    • 方法功能: 根据HTML标签名称获取页面元素集合
    • 返回值特点:
      • 返回的是伪数组（类数组对象），不是真正的JavaScript数组
      • 示例中获取所有div标签时返回了219个元素的集合
    • 注意事项:
      • 伪数组不具备标准数组的全部方法（如sort方法不可用）
      • 可以通过Array.prototype.slice.call(divs)转换为真数组
      • 转换后可以使用所有数组方法（如instanceof Array验证为true）
    • 实际应用:
      • 获取页面所有特定标签元素（如获取所有 <div>）
      • 示例中演示了获取淘宝页面所有div元素的过程
  • DOM元素获取方式对比
    • 与其他方法的区别:
      • 与getElementById()相比：后者通过id获取单个元素，而前者获取元素集合
      • 与querySelectorAll()相比：后者使用CSS选择器语法，功能更强大
    • 链式调用:
      • 可以在DOM元素上调用：dom.getElementsByTagName()
      • 这样只会在该元素的子节点中查找
  • 伪数组特性
    • 本质说明:
      • DOM集合就是普通对象，与界面元素一一对应
      • 没有特殊性质，遵循JavaScript对象的基本规则
    • 转换技巧:
      • 使用数组原型方法转换：Array.prototype.slice.call(divs)
      • 转换后可获得完整数组功能（如map、filter等方法）
3）根据元素类样式获取DOM ﻿10:16
  • 基本获取方法
    • 作用范围：获取方法分为文档级和元素级，文档级从整个网页查找，元素级只在指定DOM内部查找
    • ID获取：document.getElementById()通过元素id获取单个DOM节点
    • 标签名获取：document.getElementsByTagName()通过标签名获取DOM伪数组
    • 类名获取：document.getElementsByClassName()通过类样式获取DOM伪数组
  • 作用范围示例
    • 局部查找：在特定DOM内查找时，如dom.getElementsByTagName(‘div’)只在当前DOM内部查找，示例中仅找到25个div，远少于全局查找结果
    • 全局查找：使用document对象调用方法时，查找范围是整个网页文档
  • 特殊元素获取
    • CSS选择器：
      • document.querySelector()获取第一个匹配的DOM
      • document.querySelectorAll()获取所有匹配的DOM伪数组
    • 文档结构：
      • document.documentElement获取html元素
      • document.body获取body元素
      • document.head获取head元素
    • DOM关系：
      • dom.children获取子元素伪数组
      • dom.childNodes获取子节点伪数组
      • dom.previousElementSibling获取前一个兄弟元素
  • 方法比较
    • 历史演变：传统方法（getElementById等）功能有限，jQuery曾作为补充方案出现
    • 现代方案：Web API新增的querySelector系列方法功能更强大，已逐渐取代jQuery的DOM操作功能
    • 选择建议：根据实际需求选择方法，没有唯一标准方案
4）根据CSS选择器获取DOM ﻿12:57
  • querySelector方法：根据CSS选择器获取DOM，返回第一个匹配的元素。如document.querySelector(‘#id’)通过ID选择器获取元素，若未找到则返回null。
  • querySelectorAll方法：根据CSS选择器获取所有匹配的DOM，返回NodeList伪数组。如document.querySelectorAll(‘div’)获取所有div元素。
  • 与传统方法对比：
    • 完全替代getElementById、getElementsByClassName和getElementsByTagName
    • jQuery因这两个方法的出现而逐渐退出历史舞台
    • 支持在指定DOM内查询，如dom.querySelector()
  • ID选择器示例：document.querySelector(‘#J_SiteNavBd’)通过井号(#)前缀获取指定ID元素
  • 类选择器示例：document.querySelector(‘.J_Cover’)通过点(.)前缀获取指定类名元素
  • 层级选择器示例：document.querySelector(‘.J_Cover .site-nav’)可获取嵌套结构的元素
  • 伪数组特性：querySelectorAll返回的NodeList具有数组特性但不是真数组
  • 元素数量验证：通过淘宝页面实测，document.querySelectorAll(‘div’)可获取219个div元素
  • documentElement属性：通过document.documentElement获取HTML根元素（非方法调用）
  • 其他常用属性：
    • document.body获取body元素
    • document.head获取head元素
    • 这些属性都是直接访问而非方法调用

6）DOM元素关系操作 ﻿16:08
  • 子元素获取
    • children属性:
      • 获取当前DOM元素的所有子元素，返回HTMLCollection伪数组
      • 例如document.body.children可以获取body下的33个子元素
      • 每个子元素本身也是DOM对象，可以继续访问其children属性
    • childNodes属性:
      • 与children的区别：childNodes获取所有子节点（包括文本节点、注释节点等）
      • 举例：对于 <p>元素，children只获取 <strong>元素，而childNodes会获取3个节点（文本+元素+文本）
  • 兄弟元素获取
    • previousElementSibling: 获取当前元素的前一个兄弟元素
    • nextElementSibling: 获取当前元素的后一个兄弟元素
    • parentElement: 获取当前元素的父元素
  • 注意事项
    • DOM对象本质: DOM是JavaScript中对应HTML节点的对象模型
    • 伪数组特性: children和childNodes返回的都是类数组对象，可以通过索引访问但非真正数组
    • 链式操作: 可以连续调用属性形成链式操作，如document.body.children[0].children

5. DOM样式 ﻿42:38

﻿

1）样式类型
  • 内联样式：
    • 通过元素的style属性直接设置的样式
    • 示例：<div style="color:red">中的样式
  • 计算样式：
    • 最终应用到元素上的所有样式计算结果
    • 包含继承、层叠等CSS规则处理后的值
    • 示例：即使设置width:100%，实际计算值可能是744.5px
2）样式操作
  • 获取样式：
    • 内联样式：element.style.property
    • 计算样式：getComputedStyle(element).property
  • 修改样式：
    • 只能修改内联样式：element.style.color = ‘blue’
    • 命名转换：CSS短横线命名→JS驼峰命名（font-size → fontSize）
  • 特殊操作：
    • 删除样式：赋值为空字符串element.style.color = ‘’
    • 无效赋值：不会报错但会被忽略
    • class操作：使用className而非class（避免关键字冲突）

二、英雄展示 ﻿52:10

﻿

1. 英雄元素结构
  • 基础结构: 每个英雄由a元素构成，内部包含图片和文本两个子元素
  • 静态与动态区别: HTML中写的是静态占位样式，实际需要通过JS动态生成真实内容
  • 数据来源: 英雄数据存储在JS文件的数组中，每个英雄对象包含编号(id)和名字(name)两个关键属性
2. 动态生成实现
  • 核心思路: 通过循环遍历英雄数组，为每个对象创建对应的DOM元素
  • 创建流程:
    • 创建a元素：document.createElement(‘a’)
    • 设置a元素属性：
      • href: 格式为”英雄ID.html”（使用英雄的id属性）
      • target: 设置为”_blank”在新窗口打开
      • className: 设置样式类名
    • 创建img元素并设置src属性：图片路径格式为”英雄ID.png”
    • 创建span元素并设置innerText为英雄的name属性
    • 将img和span依次添加到a元素中
    • 将a元素添加到页面容器(container)中
3. 代码实现要点
  • DOM操作顺序: 必须先创建元素再设置属性，最后才添加到文档树中
  • 属性设置注意:
    • class属性需使用className而非class
    • 路径拼接需注意字符串格式
  • 性能考量: 虽然代码量较大但逻辑简单，复杂度主要来自DOM操作步骤
  • 替代方案: 可以使用字符串拼接+innerHTML方式简化代码（但本示例采用标准DOM操作方式）
4. 实现效果
  • 最终结构: 每个英雄显示为可点击的卡片，包含英雄图片和名称
  • 交互功能: 点击英雄卡片会跳转到对应英雄详情页（王者荣耀官网）
  • 样式表现: 通过className实现不同英雄的样式区分
