js-promise第三讲

一、邓哥的新问题

1. 每个任务可以看做是一个返回Promise的函数

  • 任务定义：每个异步任务都可以封装成一个返回Promise对象的函数，通过resolve和reject来处理成功和失败的情况
  • 实现方式：
    • 使用new Promise()构造函数创建Promise对象
    • 在异步操作完成后调用resolve或reject
    • 通过setTimeout模拟异步操作的延迟
1）具体任务示例

  • 做饭任务

    • 函数定义：
    • 特点：
      • 延迟2秒执行
      • 有50%概率成功或失败
      • 成功返回”饭已ok”，失败返回错误信息

  • 洗衣服任务

    • 函数定义：
    • 特点：
      • 延迟2.5秒执行
      • 同样有50%成功概率
      • 失败情况返回特定错误信息

  • 打扫卫生任务

    • 函数定义：
    • 特点：
      • 延迟3秒执行
      • 成功失败概率相同
      • 失败情况返回幽默的错误信息
2）任务并行执行

  • 核心需求：让三个任务同时进行，并汇总最终结果
  • 实现思路：
    • 使用Promise.all()方法并行执行多个Promise
    • 所有任务完成后统一处理结果
    • 任一任务失败都会导致整体失败
  • 关键点：
    • 并行执行可以节省总执行时间
    • 需要处理可能的失败情况
    • 结果汇总需要等待最慢的任务完成
2. 同时运行任务

  • 实现方式：直接连续调用返回Promise的函数即可实现并行执行，如cook(); wash(); sweep();
  • 特点：各任务独立运行，互不干扰，无需等待前一个任务完成
  • 示例说明：
    • 做饭函数cook()：有50%概率返回”做饭成功”或”做饭失败”
    • 洗衣服函数wash()：有50%概率返回”衣服已洗好”或”洗衣服时停水了”
    • 打扫函数sweep()：有50%概率返回”地板扫干净”或”机器人被掀翻”
3. 串行任务

  • 实现方式：通过.then()方法链式调用，如cook().then(()=>wash()).then(()=>sweep())
  • 执行顺序：
    • 先执行cook()做饭
    • 完成后执行wash()洗衣服
    • 最后执行sweep()打扫卫生
  • 代码特点：
    • 每个.then()中返回新的Promise
    • 前一个任务完成后才会触发下一个任务
    • 可通过折叠函数定义保持代码整洁
4. 汇总问题

  • 问题描述：当多个Promise并行执行时，难以统一获取所有任务的结果状态
  • 具体表现：
    • 直接在调用后写处理代码只能获取单个任务状态
    • 无法确定所有任务的完成时机
    • 缺乏统一的错误处理机制
  • 需求场景：邓哥需要等所有家务完成后，统一向邓嫂汇报哪些成功/失败
  • 现有方案缺陷：
    • 简单并行调用无法收集全部结果
    • 需要更高级的Promise组合方法解决
5. Promise的静态方法
  • 静态方法定义：通过Promise构造函数直接调用的方法，不需要实例化即可使用
  • 核心作用：提供多种处理异步任务的组合方式，满足不同场景需求
1）Promise.resolve

  • 功能：直接返回一个已完成状态的Promise对象
  • 参数处理：接收的数据会作为Promise的完成值
  • 等效写法：Promise.resolve(1) 等价于 new Promise(resolve => resolve(1))
  • 应用场景：主要用于测试和框架开发，日常业务代码使用较少
2）Promise.reject

  • 功能：直接返回一个已拒绝状态的Promise对象
  • 参数处理：接收的参数会作为Promise的拒绝原因
  • 等效写法：Promise.reject(1) 等价于 new Promise((_, reject) => reject(1))
  • 错误处理：未捕获的reject会导致UnhandledPromiseRejectionWarning警告
3）Promise.all

  • 执行机制：接收Promise数组，所有Promise成功时整体成功，任一失败立即整体失败

  • 成功结果：返回数组包含所有Promise的完成值，顺序与输入一致

  • 失败特点：遇到第一个失败即终止，只返回该失败原因

  • 挂起情况：只要有一个Promise未决，整体保持pending状态

  • 例题:Promise.all应用

    • 成功案例：三个立即完成的Promise，返回[1,2,3]数组
    • 失败案例：包含一个reject时，立即返回该reject原因
    • 挂起案例：包含pending状态的Promise时，整体保持pending
4）Promise.any

  • 执行机制：接收Promise数组，任一成功即整体成功，全部失败才整体失败

  • 成功结果：返回第一个成功的Promise的值（非数组）

  • 失败处理：全部失败时返回AggregateError对象，通过errors属性获取所有原因

  • 版本注意：需较高版本的Node.js或浏览器环境支持

  • 例题:Promise.any应用

    • 成功案例：包含resolve(1)时返回1，忽略其他Promise状态
    • 失败案例：全部reject时返回包含所有原因的AggregateError
    • 浏览器测试：在Chrome控制台可正常运行，Node低版本会报错
5）Promise.allSettled

  • 执行机制：等待所有Promise完成（无论成功失败），不会reject

  • 结果格式：返回对象数组，每个对象包含status和value/reason

  • 应用场景：需要获取所有异步操作最终状态的场景

  • 与all区别：不关心成功/失败，只关心是否已决

  • 例题:Promise.allSettled应用

    • 混合状态：成功返回{status:’fulfilled’,value}，失败返回{status:’rejected’,reason}
    • 结果顺序：保持与输入Promise相同的顺序
    • 挂起情况：有未决Promise时整体保持pending
6）Promise.race

  • 执行机制：接收Promise数组，采用第一个改变状态的Promise的结果

  • 结果特点：完全跟随第一个完成的Promise（无论成功/失败）

  • 应用场景：超时控制、竞速场景

  • 注意事项：其他Promise的结果会被忽略但不会取消

  • 例题:Promise.race应用

    • 成功优先：第一个resolve的Promise决定整体成功状态和值
    • 失败优先：第一个reject的Promise决定整体失败状态和原因
    • 实际应用：异步操作中取最快响应的结果
6. Promise静态方法应用案例
1）获取学生数据函数

  • 函数功能：模拟网络请求获取分页学生数据
  • 参数说明：接收页码参数page，返回Promise对象
  • 返回值特点：
    • 成功时返回学生数组，包含id和name属性
    • 失败时返回错误信息”网络错误！获取第X页数据失败！”
    • 使用随机数模拟网络延迟和失败概率
2）Promise.all应用

  • 使用场景：需要获取1-10页数据，全部成功才继续处理
  • 实现要点：
    • 创建包含10个fetchStudents调用的promise数组
    • 使用Promise.all(proms).then().catch()处理结果
    • 结果处理：
      • 成功时用result.flat()合并多维数组
      • 失败时立即终止并打印错误
  • 数组创建技巧：
    • 使用new Array(10).fill(1).map((_,i)=>fetchStudents(i+1))简化代码
    • 保证结果顺序与传入顺序一致
3）Promise.allSettled应用

  • 使用场景：允许部分请求失败，只合并成功数据
  • 实现步骤：
    • 过滤status === ‘fulfilled’的结果
    • 映射提取value属性
    • 使用flat()拍平数组
  • 与all的区别：
    • all遇到第一个失败就终止
    • allSettled会等待所有promise完成
    • allSettled返回对象包含status字段
4）Promise.any应用

  • 使用场景：获取最先返回的成功结果
  • 错误处理：
    • 全部失败时通过err.errors获取所有错误
    • 错误信息包含具体失败页码
  • 注意事项：
    • 需要现代浏览器或Node.js高版本支持
    • 与race的区别：any只关注成功结果
5）Promise.race应用

  • 使用场景：获取最先完成的promise（无论成功/失败）
  • 实现方式：
    • 直接使用Promise.race(proms)
    • 通过then/catch处理最先返回的结果
  • 典型应用：
    • 请求超时控制
    • 竞速获取最快响应

二、Promise的静态方法

1. 邓哥的解决方案

  • 方法对比:
    • Promise.resolve(data): 直接返回完成状态的Promise
    • Promise.reject(reason): 直接返回拒绝状态的Promise
    • Promise.all(): 全部成功则成功，任一失败则失败
    • Promise.any(): 任一成功则成功，全部失败则失败
    • Promise.allSettled(): 全部已决则成功（不会失败）
    • Promise.race(): 任一已决则已决，状态与其一致
  • 实际应用:
    • Promise.race()示例：获取1-10页学生数据，输出最先返回的结果（无论成功/失败）
    • Promise.any()示例：获取1-10页学生数据，打印最先成功的数据，全部失败才打印错误
  • 邓哥案例解析:
    • 场景：需要汇总做饭、洗衣、打扫三个异步任务的结果
    • 解决方案：
    • 实现要点:
      • 使用allSettled确保获取所有任务最终状态
      • 通过map处理结果：成功取value，失败取reason
      • 用join将结果拼接成字符串报告
  • 调试技巧:
    • 可先直接打印result观察原始数据结构
    • 示例输出可能包含：
      • 成功结果：{status: “fulfilled”, value: …}
      • 失败结果：{status: “rejected”, reason: …}
  • 适用场景:
    • race：竞速场景，如超时控制
    • any：乐观获取，容忍部分失败
    • allSettled：必须获取完整报告的场景
    • all：强一致性要求场景

三、知识小结

知识点 核心内容 考试重点/易混淆点 难度系数
Promise静态方法 resolve/reject快速创建成功/失败的Promise对象 直接返回状态确定的Promise ⭐⭐
Promise.all 全成功才成功，失败立即终止，返回结果数组 失败时仅返回第一个错误原因 ⭐⭐⭐⭐
Promise.any 任一成功即成功，全部失败才返回错误数组 与race的区分（不论成功失败） ⭐⭐⭐⭐
Promise.allSettled 等待所有结果，返回包含状态和值的详细数组 结果对象结构（status/value/reason） ⭐⭐⭐
Promise.race 竞速获取首个结果（无论成功失败） 应用场景选择（超时控制等） ⭐⭐⭐
多任务控制模式 串行vs并行执行对比（做饭/洗衣案例） then链式调用与并行调用的代码差异 ⭐⭐⭐⭐
结果聚合处理 flat映射展开嵌套数组，filter过滤状态 错误处理时的数据完整性保持 ⭐⭐⭐⭐
实战案例 分页数据抓取（1-10页并行请求） all与allSettled的容错性差异 ⭐⭐⭐⭐
