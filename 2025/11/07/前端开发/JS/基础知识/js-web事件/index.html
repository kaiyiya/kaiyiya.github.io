一、事件

1. 例题：注册点击事件
1）事件注册基础
  • 事件三要素：
    • 监听对象：通过document.querySelector()获取DOM元素
    • 事件类型：如click表示点击事件
    • 处理函数：定义事件触发时执行的操作
  • 注册方法：
    • 使用addEventListener方法
    • 或直接给DOM元素的onclick属性赋值函数
    • 示例代码：
2）事件处理机制
  • 执行时机：
    • 事件处理函数不会立即执行
    • 只有当指定事件（如点击）发生时才会触发
    • 示例中console.log(‘点击’)只在按钮被点击时输出
  • 函数内容：
    • 处理函数内部可以包含任意合法JS语句
    • 不仅限于简单的输出语句
3）函数定义方式
  • 定义方式：
    • 匿名函数表达式：直接在事件注册时定义
    • 命名函数：先定义函数再赋值给事件属性
    • 两种方式效果完全相同
  • 代码示例：
4）事件注册执行流程
  • 同步特性：
    • 事件注册是同步操作，会立即完成
    • 注册语句之后的代码会继续执行
    • 示例中console.log(‘end’)会立即输出
  • 异步触发：
    • 事件处理函数是异步执行的
    • 只有在事件发生时才会被调用
    • JS引擎不会等待事件发生
5）事件模型核心概念
  • 事件三要素：
    • 监听目标：明确要监听的DOM元素
    • 事件类型：确定要监听的具体事件（如click）
    • 处理函数：定义事件触发时的响应逻辑
  • 通用模式：
  • 扩展性：
    • 不同事件只需改变事件类型（如mousemove、keydown等）
    • 基本注册机制保持不变
2. 事件类型
  • 表单类事件：主要应用于表单元素交互
  • 鼠标类事件：处理鼠标相关操作
  • 键盘事件：响应键盘操作
  • DOM事件：处理DOM结构变化
1）表单类事件

  • submit：表单提交时触发，注册到form元素

  • input：文本框内容改变时立即触发，适用于input/textarea

  • change：适用于input/select/textarea，下拉列表/多选框/单选框改变时触发

  • focus：元素获得焦点时触发

  • blur：元素失去焦点时触发

  • reset：表单重置时触发

  • 例题:注册submit事件

    • 实现方式：
      • 通过document.querySelector(‘form’)获取表单元素
      • 使用onsubmit属性注册事件处理函数
    • 特点：
      • 表单提交会导致页面刷新
      • 适用于表单提交前的数据验证等场景
    • 注意事项：
      • 应注册到form元素而非按钮
      • 回车键和点击按钮都会触发
2）input事件

  • 触发时机：文本框内容每次改变时立即触发

  • 应用场景：实时搜索建议、输入验证等

  • 获取值：通过元素的value属性获取当前输入内容

  • 例题:监听input事件

    • 实现步骤：
      • 获取input元素：document.querySelector(‘input’)
      • 注册事件：oninput = function(){}
      • 在事件处理函数中通过value属性获取当前值
    • 特点：
      • 每次按键都会触发
      • 适用于需要实时响应输入的场景
3）change事件

  • 与input事件区别：

    • input：每次改变立即触发
    • change：失去焦点且内容改变时触发

  • 在下拉列表中：选择选项时立即触发，无需失去焦点

  • 例题:change事件应用

    • 实现方式：
      • 获取select元素：document.querySelector(‘select’)
      • 注册事件：onchange = function(){}
    • 典型应用：
      • 监听下拉列表选择变化
      • 表单字段修改后的验证
4）鼠标类事件
  • click：鼠标按下并抬起后触发
  • contextmenu：右键菜单显示前触发
  • mousedown：鼠标按下时触发
  • mouseup：鼠标抬起时触发
  • mousemove：鼠标在元素上移动时触发
  • mouseenter/leave：鼠标进入/离开元素时触发（不冒泡）
  • mouseover/out：鼠标进入/离开元素时触发（冒泡）
  • wheel：鼠标滚轮滚动时触发
5）键盘事件
  • keydown：按键按下时触发
  • keyup：按键抬起时触发
  • 注意事项：
    • keypress事件已被弃用
    • 可以注册到window对象监听全局按键
    • 通过事件对象可以获取具体按键信息
3. 注册事件
1）方式2：使用dom属性注册事件

      • 只能注册一个处理函数，后注册的会覆盖之前的
      • 某些新版本标准的事件不支持这种方式注册

    • 可以注册多个事件处理函数，不会互相覆盖
    • 支持所有标准事件类型
    • 需要单独移除每个事件处理函数
  • 移除事件：

    • 避免在事件处理函数中直接写匿名函数，否则无法移除

1）事件对象详解
  • 参数特性：
    • 事件对象参数名称可自定义（常用e/event），但实际传递的是包含事件数据的对象
    • 该参数包含如点击位置（clientX/clientY）、按键状态等上下文信息
  • 使用场景：
    • 需要获取事件细节时使用，如开发需要知道点击坐标时
    • 不需要时可忽略该参数，不影响基础功能

5. 鼠标事件
  • 核心对象：MouseEvent对象包含用户与指针设备交互的所有信息，常见事件包括click、dblclick、mouseup等
  • 重要属性：
    • clientX/clientY：鼠标相对于视口的坐标
    • pageX/pageY：鼠标相对于文档的坐标
    • offsetX/offsetY：鼠标相对于目标元素内边距的坐标
    • button：标识按下的鼠标按键（0=左键，1=中键，2=右键）
  • 开发参考：MDN的MouseEvent文档包含完整属性列表和方法说明，是开发时的重要参考资料
6. 键盘事件

  • 查阅文档：需要获取键盘事件信息时，应查阅MDN的KeyboardEvent文档，该文档以列表形式详细说明了键盘事件对象的可用属性及其含义。

  • KeyboardEventInit字典：

    • 基础属性：
      • key：DOMString类型，默认空字符串，对应KeyboardEvent.key
      • code：DOMString类型，默认空字符串，对应KeyboardEvent.code
      • location：unsigned long类型，默认0，对应KeyboardEvent.location
    • 修饰键状态：
      • ctrlKey/shiftKey/altKey/metaKey：Boolean类型，默认false，分别对应各修饰键状态
    • 其他属性：
      • repeat：Boolean类型，表示按键是否重复
      • isComposing：Boolean类型，表示是否在输入法组合过程中
    • 废弃属性（应避免使用）：
      • charCode/keyCode/which：均为unsigned long类型，默认0
7. 事件处理机制
  • 事件对象：
    • 自动传递：事件发生时浏览器会自动向处理函数传递事件对象参数
    • 包含信息：如鼠标位置（e.clientX）、键盘按键等特定事件的相关数据
    • 常见类型：分为鼠标事件对象和键盘事件对象两大类
  • this指向规则：
    • 固定指向：事件处理函数中的this始终指向注册事件的DOM元素
    • 与变量解耦：即使将保存DOM的变量（如btn）置为null，this仍保持原始引用
    • 优势：比直接使用变量引用更可靠，避免因变量重新赋值导致的问题
8. 事件监听核心要点
  • 学习纲领：
    • 核心原则：掌握”哪个DOM发生什么事件后执行什么处理”的基本模型
      • 监听目标（某个DOM）
      • 事件类型（某件事）
      • 处理逻辑（某些处理）
    • 细节处理：具体事件类型、处理函数参数等应通过查阅文档获取
    • 示例代码：
  • 事件注册方式：
    • onclick方式：
      • 优点：简单易用，支持覆盖和移除（设为null）
      • 缺点：单个事件只能绑定一个处理函数
    • addEventListener：
      • 优势：支持多个处理函数绑定
      • 移除方法：需使用removeEventListener并传入相同函数引用

二、应用案例

1. 例题：用户注册表单事件应用
1）表单初始化状态设置
  • 函数调用顺序：页面加载时需要依次调用三个状态设置函数，确保初始状态正确：
    • setSubmitButtonStatus()：设置注册按钮状态
    • setSendCodeButtonStatus()：设置验证码发送按钮状态
    • setSelectedLoves()：设置已选爱好显示状态
  • 开发习惯：建议先编写功能函数再处理事件绑定，使代码结构更清晰
2）事件绑定实现

  • 验证码按钮状态控制

    • 绑定方式：通过addEventListener监听手机号文本框的input事件
    • 事件处理：直接传入setSendCodeButtonStatus函数作为回调
    • 验证逻辑：当文本框长度不等于11时禁用按钮（btn.disabled = txt.value.length != 11）

  • 协议多选框状态控制

    • 事件类型：使用change或input事件监听选中状态变化
    • 状态判断：通过$(‘.policy input’).checked获取多选框选中状态
    • 简写技巧：直接将setSubmitButtonStatus函数作为事件回调

  • 爱好选择状态更新

    • 事件绑定：监听select元素的change事件
    • 数据处理：使用join()方法将选中的爱好用逗号连接显示
    • 动态更新：每次选择变化时重新生成显示文本
3）ESC键清空文本框实现
  • 元素获取：使用document.querySelectorAll(‘.txt’)获取所有文本框
  • 事件监听：为每个文本框添加keydown事件监听
  • 按键判断：通过事件对象的key属性判断是否为Escape键
  • 清空操作：使用this.value = ‘’清空当前触发事件的文本框
  • 开发技巧：
    • 可先将事件处理逻辑提取为独立函数handleKeyDown
    • 通过console.log调试事件对象和this指向
4）事件处理最佳实践
  • 分离原则：功能逻辑与事件处理分离，先实现功能函数再处理事件绑定
  • 事件代理：相似元素的事件处理可使用事件代理优化性能
  • 调试方法：通过打印事件对象和this值验证事件处理逻辑
  • 代码组织：相关事件处理函数集中放置，添加清晰注释说明功能
2. 例题：图片自动切换功能实现
1）功能实现原理
  • 核心机制：通过setInterval定时器实现图片自动切换，使用currentIndex变量记录当前显示图片的序号（1-4）
  • 切换逻辑：每隔1秒修改图片的src属性路径，将数字部分替换为currentIndex的值
  • 循环处理：当currentIndex超过4时重置为1，或使用取模运算currentIndex % 4 + 1实现循环
2）代码实现要点
  • 元素获取：使用document.querySelector获取图片DOM元素，建议开发时先打印确认
  • 计时器管理：
    • 定义timerId变量存储计时器ID
    • 在start函数中先检查timerId是否存在，避免重复创建计时器
    • 停止时需调用clearInterval并重置timerId为null
3）交互功能实现
  • 自动启动：页面加载后立即调用start函数
  • 鼠标事件：
    • mouseenter事件触发stop函数
    • mouseleave事件触发start函数
  • 开发习惯建议：先独立编写功能函数（如start/stop），再绑定事件处理
4）注意事项
  • 计时器累积：防止多次调用start导致多个计时器并行运行
  • 变量清理：停止计时器后必须将timerId置为null
  • 边界处理：图片索引需要在1-4之间循环
3. 例题：代办事项小程序
1）程序功能概述
  • 核心功能：文本框输入内容回车后添加到待办列表，点击删除按钮移除对应项
