js-dom的尺寸和位置

飞向月球的猫2025-11-102025-11-20

一、DOM尺寸和位置

1. 获取屏幕尺寸
  • 屏幕尺寸获取：
    • window.screen.height：获取整个屏幕的高度
    • window.screen.width：获取整个屏幕的宽度
  • 浏览器窗口尺寸获取：
    • window.outerHeight：获取包含浏览器边框和工具栏的完整高度
    • window.outerWidth：获取包含浏览器边框和工具栏的完整宽度
  • 视口尺寸获取：
    • window.innerHeight：获取浏览器内部高度（包含滚动条）
    • window.innerWidth：获取浏览器内部宽度（包含滚动条）
    • document.documentElement.clientHeight：获取视口高度（不包含滚动条）
    • document.documentElement.clientWidth：获取视口宽度（不包含滚动条）
2. 获取元素尺寸
  • client系列：
    • element.clientWidth：获取元素内容+padding的宽度（不包含边框和滚动条）
    • element.clientHeight：获取元素内容+padding的高度（不包含边框和滚动条）
  • offset系列：
    • element.offsetWidth：获取元素内容+padding+边框的宽度（包含滚动条）
    • element.offsetHeight：获取元素内容+padding+边框的高度（包含滚动条）
  • scroll系列：
    • element.scrollWidth：获取元素实际内容的宽度（包含不可见部分）
    • element.scrollHeight：获取元素实际内容的高度（包含不可见部分）
    • element.scrollTop：获取元素垂直滚动条滚动的距离
    • element.scrollLeft：获取元素水平滚动条滚动的距离
3. 元素滚动控制
  • 滚动位置设置：
    • element.scrollTo(x, y)：设置元素的滚动位置，x表示水平滚动距离，y表示垂直滚动距离
    • element.scrollTo(0, 0)：将元素滚动到顶部最左侧
  • 滚动事件监听：
    • 监听scroll事件可以获取元素的滚动状态
    • 使用addEventListener(‘scroll’, callback)添加滚动监听
4. 元素位置获取
  • getBoundingClientRect()：
    • 返回元素相对于视口的位置信息对象
    • 包含top、left、right、bottom等属性
    • 所有位置值都是相对于视口的坐标
5. 获取元素位置
1）getBoundingClientRect方法
  • 方法调用：通过元素对象的getBoundingClientRect()方法获取元素边界矩形
  • 边界定义：包含元素整个盒模型（content+padding+border）在内的矩形区域
  • 翻译说明：bounding意为”边界”，client指”客户端/视口”，rect是”矩形”的缩写
2）矩形对象的信息
  • 基本属性：
    • width/height：元素的完整宽度和高度（含边框）
    • x/y：矩形左上角相对于视口的坐标（等同于left/top）
  • 位置属性：
    • left：元素左边界到视口左边的距离（单位：像素）
    • top：元素上边界到视口顶部的距离
    • right/bottom：元素右/下边界到视口左/顶边的距离（使用频率较低）
3）相对于视口的位置
  • 视口概念：当前浏览器窗口可见区域（与固定定位的参照系相同）
  • 实际应用：通过京东购物车示例，left=993.5表示元素左边界距离视口左边993.5像素
  • 注意事项：宽高信息通常使用其他专门方法获取，不推荐通过此方式
4）相对于父元素的位置
  • 计算原理：
    • 获取元素离视口左边距离（elementRect.left）
    • 获取父元素离视口左边距离（parentRect.left）
    • 两者相减得到相对位置
  • 数学表达：相对位置 =
    elementRect.left−parentRect.leftelementRect.left - parentRect.leftelementRect.left**−parentRect.left**
  • 实现建议：需要时查阅API文档即可，不必死记硬背方法
6. 应用案例

  • 功能实现分析

    • 基本功能：
      • 顶部移动条可拖动整个便签
      • 内容区域可编辑文字
      • 鼠标悬停在移动条时显示移动光标样式
    • 实现原理：
      • 内容可编辑：通过设置contenteditable属性实现
      • 移动条样式：使用CSS设置cursor: move
      • 拖动限制：不能超出浏览器视口范围

  • HTML结构

    • .note：便签容器，设置为position: fixed
    • .move-bar：拖动控制条
    • .content：可编辑内容区域

      • 开始监听整个窗口的鼠标移动（mousemove）
      • 鼠标抬起时（mouseup）取消所有监听

2）例题:自行运动的小球

  • 小球运动实现原理

    • 基本结构：使用固定定位的小球元素，通过CSS设置宽高50px、圆形边框和初始位置
    • 运动机制：通过JavaScript每隔20毫秒改变小球的left和top值实现运动效果
    • 方向控制：通过正负值控制运动方向，正值表示右下运动，负值表示左上运动

  • 核心代码实现

    • 定时器设置：使用setInterval函数，设置20毫秒的间隔时间
    • 元素获取：在外部通过document.querySelector获取小球元素，避免重复获取
    • 坐标计算：通过getBoundingClientRect()获取当前坐标，加上位移值得到新坐标

  • 边界检测与反弹

    • 屏幕尺寸获取：使用document.documentElement.clientWidth/Height获取可视区域尺寸
    • 小球尺寸获取：通过ball.clientWidth/Height获取小球自身尺寸
    • 最大坐标计算：最大left=屏幕宽度-小球宽度，最大top=屏幕高度-小球高度
    • 反弹逻辑：当坐标超过边界时，将位移值取反(disX = -disX)

  • 碰撞效果增强

    • 随机数生成：使用Math.random()生成0-255之间的随机RGB值
    • 颜色改变：碰撞时调用changeBg()函数，设置ball.style.background为新RGB值
    • 颜色优化：可将随机范围限制在0-200之间，避免颜色过淡看不清

  • 完整实现代码

    • 变量定义：包含位移值(disX/disY)、屏幕尺寸(w/h)、小球尺寸(ew/eh)
    • 核心循环：定时器中计算新坐标、检测边界、处理反弹和颜色变化
    • 性能优化：所有不变的元素和尺寸都在循环外获取，避免重复计算

  • 扩展思考

    • 运动速度：可通过调整disX/disY的值改变运动速度
    • 运动方向：初始值可设为负数实现不同方向的初始运动
    • 碰撞检测：可扩展为多个小球之间的碰撞检测和反弹
    • 重力模拟：可加入加速度模拟重力效果

二、知识小结

知识点 核心内容 考试重点/易混淆点 难度系数
DOM尺寸获取 通过window.outerWidth/Height获取浏览器尺寸，window.innerWidth/Height获取包含滚动条的视口尺寸，document.documentElement.clientWidth/Height获取不包含滚动条的视口尺寸 区分浏览器尺寸和视口尺寸，注意滚动条是否包含在内 ⭐⭐
元素尺寸获取 clientWidth/Height获取内容+padding（不含边框和滚动条），offsetWidth/Height获取包含边框和滚动条的尺寸 滚动条和边框的包含关系，不同场景选择合适API ⭐⭐⭐
滚动位置控制 scrollTop/Left获取滚动距离，scrollTo(x,y)设置滚动位置，通过scroll事件监听滚动 理解坐标系原点（左上角为0,0），横向/纵向滚动差异 ⭐⭐
元素位置获取 getBoundingClientRect()获取元素相对于视口的矩形信息（含left/top/right/bottom） 相对视口而非父元素，需手动计算与父元素距离 ⭐⭐⭐⭐
拖拽实现原理 1. 鼠标按下记录初始坐标; 2. 移动时计算偏移量; 3. 限制边界防止超出视口 事件监听顺序（mousedown→mousemove→mouseup），坐标转换计算 ⭐⭐⭐⭐
碰撞反弹效果 1. 定时更新元素位置; 2. 边界检测时反转移动方向; 3. 碰撞时触发附加效果（如变色） 运动方向向量控制，边界条件判断逻辑 ⭐⭐⭐
性能优化要点 1. 避免在频繁触发的事件中查询DOM; 2. 提前缓存不变的值（如视口尺寸）; 3. 使用CSS硬件加速 重排/重绘代价，getBoundingClientRect的性能影响 ⭐⭐⭐⭐
