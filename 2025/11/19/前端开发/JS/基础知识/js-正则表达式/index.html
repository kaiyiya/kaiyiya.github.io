一、标准库

1. 正则表达式
1）正则常用方法

  • 创建正则对象

    • 构造函数创建：
      • 语法：new RegExp(‘规则’,’标识’)
      • 第一个参数是字符串形式的匹配规则，第二个参数是可选的标识符（如’i’表示不区分大小写）
      • 示例：var reg = new RegExp(‘a’, ‘i’) 表示创建匹配字母a且不区分大小写的正则对象
    • 字面量创建：
      • 语法：/规则/标识
      • 更简洁的创建方式，规则写在两个斜杠之间，标识符跟在最后一个斜杠后
      • 示例：var reg = /a/i 效果与构造函数方式相同

  • 正则对象常用方法

    • test方法：
      • 语法：reg.test(‘字符串’)
      • 功能：验证字符串是否满足正则规则，返回布尔值
      • 示例：reg.test(‘234234324a234234234’) 检查字符串是否包含a，返回true
    • replace方法：
      • 语法：str.replace(reg,’替换目标’) 或 str.replace(reg, function(s){return’替换目标’})
      • 功能：将字符串中匹配正则的部分替换为目标内容
      • 回调函数形式可以将匹配内容传入函数处理后再替换
      • 示例：将字符串中所有1替换为a：str.replace(/1/g, ‘a’)

  • 正则标识符

    • 常用标识符：
      • i：不区分大小写匹配
      • g：全局匹配（匹配所有符合条件的内容，而非仅第一个）
      • 可以组合使用，如ig表示不区分大小写且全局匹配
    • 标识符效果：
      • 带i标识时，/a/i可以匹配字符串中的A或a
      • 不带i标识时，匹配严格区分大小写

  • 注意事项

    • 方法兼容性：
      • replaceAll方法在某些环境下可能不可用，建议使用带g标识的replace方法替代
      • 示例错误：str.replaceAll(‘1’, ‘a’)可能报错，应改为str.replace(/1/g, ‘a’)
    • 规则书写：
      • 构造函数方式中，规则参数必须是字符串形式
      • 字面量方式中，规则直接写在斜杠之间，不需要引号
2）字符匹配规则

  • 正则表达式基础

    • 创建正则对象
      • 构造函数方式：使用new RegExp(‘规则’,’标识’)创建正则对象，如new RegExp(‘a’,’i’)
      • 字面量方式：使用/规则/标识语法创建，如/a/i，本质上是构造函数的语法糖
      • 标识符作用：控制匹配细节，如是否区分大小写等
    • 常用方法
      • test方法：reg.test(‘字符串’)验证字符串是否满足规则，返回布尔值
      • replace方法：
        • 基本形式：str.replace(reg,’替换目标’)将匹配部分替换为固定字符串
        • 回调形式：str.replace(reg,function(s){return ‘替换目标’})可动态生成替换内容
      • 全局匹配：添加g标识可匹配所有符合规则的位置，否则只匹配第一个
    • 标识规则
      • i标识：忽略大小写匹配，如/a/i可匹配”A”和”a”
      • g标识：全局匹配，匹配所有符合规则的位置
      • m标识：多行匹配（较少使用）
    • 字符匹配规则
      • 普通字符：直接书写匹配对应字符，如/a/匹配字母”a”
      • 字符集合：
        • [abc]：匹配a、b或c中任意一个字符
        • [^abc]：匹配除a、b、c外的任意字符
      • 特殊字符：
        • \d：匹配数字，等价于[0-9]
        • \D：匹配非数字
        • \s：匹配空白字符（空格、回车、换行、制表符等）
        • \S：匹配非空白字符
        • \w：匹配单词字符（字母、数字、下划线），等价于[A-Za-z0-9_]
        • \W：匹配非单词字符
        • .：匹配任意字符（除换行符外）
      • 位置匹配：
        • ^：匹配字符串开始位置
        • $：匹配字符串结束位置
    • 开发工具推荐
      • VS Code插件：Regex Previewer可实时测试正则表达式匹配效果
      • 调试技巧：插件会高亮显示匹配结果，方便检查正则表达式是否正确

  • 直接书写字符匹配

    • 基本规则：直接书写字符即可匹配对应字符，例如写”a”就匹配a，”b”匹配b，”1”匹配1。
    • 匹配特点：每次只匹配一个字符，不是匹配多个相同字符。
    • 示例：
      • 正则表达式/a/匹配字符串中的第一个a
      • 正则表达式/1/匹配字符串中的第一个1

  • 中括号内的字符规则

    • 基本规则：中括号[]内的字符表示”或”关系，匹配其中任意一个字符。
    • 匹配特点：仍然只匹配一个字符，不是匹配连续的多个字符。
    • 示例：
      • [123]匹配1或2或3
      • [a-z]匹配任意小写字母
      • [A-Z]匹配任意大写字母
      • [0-9]匹配任意数字（可简写为\d）
      • [a-zA-Z]匹配任意字母
      • [a-zA-Z0-9]匹配任意字母或数字
    • 取反规则：在中括号内使用^表示不匹配指定字符
      • [^a-z]匹配非小写字母的任意字符
      • [^0-9]匹配非数字的任意字符
    • 量词规则：
      • {n}：匹配n次
      • {n,m}：匹配n到m次
      • {n,}：至少匹配n次
      • *：匹配0次或多次（等价于{0,}）
      • ?：匹配0次或1次（等价于{0,1}）
      • +：匹配1次或多次（等价于{1,}）

  • 正则表达式常用方法

    • test方法：
      • 语法：reg.test(‘字符串’)
      • 功能：验证字符串是否满足正则规则
      • 示例：/a/.test(‘abc’)返回true
    • replace方法：
      • 基本替换：str.replace(reg,’替换目标’)
      • 回调函数替换：str.replace(reg, function(s){return ‘替换目标’})
      • 示例：将每个数字左右加上尖括号
    • 结果为”<1><2><3>”
    • 标志参数：
      • i：不区分大小写
      • g：全局匹配（匹配所有符合条件的结果）

  • 正则表达式创建方式

    • 字面量方式：
      • 语法：/pattern/flags
      • 示例：var reg = /a/i
    • 构造函数方式：
      • 语法：new RegExp(‘pattern’, ‘flags’)
      • 示例：var reg = new RegExp(‘a’, ‘i’)
    • 两种方式区别：
      • 字面量方式更简洁
      • 构造函数方式可以在运行时动态构建正则表达式

  • 匹配任意字符

    • 点号匹配规则: 在正则表达式中使用.可以匹配任意字符，包括空格、字母、汉字等所有字符
    • 特殊含义说明: 点号在正则中有特殊含义，若要匹配实际的点号字符需要转义
    • 示例对比:
      • d2 匹配任意字符
      • 若要匹配字面量点号需使用.

  • 特殊字符的匹配与转义

    • 转义规则:
      • 匹配点号需使用.，否则会被识别为通配符
      • 匹配斜杠需使用\，因为单个斜杠\在正则中表示转义符
      • 左斜杠同样需要转义，否则会导致格式错误
    • 原理说明:
      • 转义符会取消字符的特殊含义，使其表示字面量
      • 与字符串中的转义规则原理相同
    • 常见错误:
      • 直接使用/匹配斜杠会导致语法错误
      • 未转义的点号会错误匹配任意字符

  • 匹配数字与非数字

    • 数字匹配:
      • \d 等价于 [0-9]，专门用于匹配数字字符
      • 记忆口诀：”d for digit”
    • 非数字匹配:
      • \D 等价于 [^0-9]，匹配所有非数字字符
      • 记忆方法：大写D表示”非”数字
    • 符号对应关系:
      • 尖号^在中括号内表示取反
      • [^0-9]与\D功能完全相同
    • 应用提示:
      • 不需要死记硬背，理解中括号和反义符号的用法即可
      • 实际使用时选择更简洁的表达方式

  • 匹配空白字符与非空白字符

    • 空白字符匹配
      • \s的含义：匹配所有空白字符，包括空格、回车（\r）、换行（\n）、制表符（\t）等
      • 实际应用：在全局匹配模式下（使用g标志），可以匹配字符串中所有符合的空白字符
      • 匹配范围：不仅能匹配可见空格，还能匹配换行符等不可见空白字符（虽然部分编辑器可能无法直观显示）
    • 非空白字符匹配
      • \S的含义：匹配所有非空白字符，与\s的功能正好相反
      • 字符互补：\S相当于[^\s]，即匹配不在空白字符集合中的任何字符
      • 实际区别：当需要排除空格、换行等空白字符时使用\S，需要包含时使用\s
    • 相关字符匹配规则
      • \w匹配：匹配单词字符，等价于
        [A−Za−z0−9][A-Za-z0-9_][A−Za−z0**−9]**
        ，包括大小写字母、数字和下划线
      • \d匹配：匹配数字字符，等价于[0-9]
      • .匹配：匹配任意单个字符（除换行符外）
      • []用法：匹配方括号中出现的任意一个字符
      • [^]用法：匹配不在方括号中出现的任意字符

  • 匹配单词字符与非单词字符

    • \w匹配规则：匹配单词字符，等价于
      [A−Za−z0−9][A-Za-z0-9_][A−Za−z0**−9]**
      ，即：* 小写字母a到z
      • 大写字母A到Z
      • 数字0到9
      • 下划线_
    • \W匹配规则：匹配非单词字符，是\w的取反，等价于
      [A−Za−z0−9][^A-Za-z0-9_][A−Za−z0**−9]**
    • 注意事项：
      • 匹配的是单个字符而非整个单词
      • 不加全局标志g时，只会匹配第一个符合的字符
      • 例如匹配”word”时，不加g只会匹配第一个字母’w’
    • 字符集匹配：
      • [字符规则]：匹配中括号内出现的任意字符
      • [^字符规则]：匹配中括号内未出现的任意字符
    • 其他常用字符规则：
      • \d\d**\d**
        ：匹配数字，等价于[0-9]
      • \D\D**\D**
        ：匹配非数字
      • \s\s**\s**
        ：匹配空白字符（空格/回车/换行/制表符）
      • §\S**§**
        ：匹配非空白字符
    • 边界匹配：
      • ^：匹配字符串开始位置
      • $：匹配字符串结束位置
      • \：匹配反斜杠本身
    • 示例代码解析
      *
      • 代码功能：将字符串中所有数字1替换为<1>形式
      • 关键点：
        • 使用/1/g全局匹配所有数字1
        • replace方法的回调函数对匹配项进行包装
        • 输出结果：’21365aBa462aca31453qa<1>23<1>’
      • 匹配原理：
        • 正则表达式/1/g会匹配字符串中所有的’1’
        • 每次匹配到’1’时，回调函数将其替换为<1>

  • 匹配字符串的开始与结束

    • 字符串开始匹配规则
      • 位置要求：必须写在正则表达式规则开始的位置
      • 符号表示：使用 ^ 符号表示匹配字符串开始
      • 匹配原理：表示必须从字符串的第一个字符开始匹配
      • 示例说明：如正则表达式 /^d/ 表示字符串必须以数字开头，若第一个字符不是数字则匹配失败
    • 字符串结束匹配规则
      • 符号表示：使用 $ 符号表示匹配字符串结束
      • 匹配原理：表示匹配的字符必须是字符串的最后一个字符
      • 示例说明：
        • `/d$/ 可以匹配 “abc123” 中的 “3”
        • 但不能匹配 “123abc” 因为数字不在字符串末尾
    • 组合使用示例
      • 完整匹配：同时使用 ^ 和 $ 可以精确匹配整个字符串
      • 典型应用：用于验证输入格式，如手机号验证 /^1\d{10}$/
      • 注意事项：
        • 开始符号必须写在规则最前面
        • 结束符号必须写在规则最后面
        • 两者位置错误会导致匹配失败
    • 常见错误分析
      • 错误类型1：将 ^ 写在规则中间位置
        • 例如 /a^b/ 无法匹配任何字符串
      • 错误类型2：忽略字符串中间内容
        • 例如 /^abc$/ 只能匹配 “abc” 不能匹配 “abc123abc”
      • 错误类型3：错误理解量词作用范围
        • 例如 /^[a-z]{3}$/ 表示必须且只能是3个小写字母
3）连续的规则

  • 连续书写规则介绍

    • 连续匹配原理：正则表达式中单个字符规则可以连续书写，用于匹配多个连续字符，如邮箱验证等场景需要按顺序匹配多个规则。

  • 示例：匹配数字后紧跟字母

    • 基础示例：\d[a-zA-Z]表示匹配1个数字后紧跟1个字母，如匹配”3a”这样的组合。
    • 边界控制：添加^和$可限定字符串开始和结束，如^\d[a-zA-Z]\d$表示”数字+字母+数字”的完整匹配。

  • 规则的组合与匹配

    • 组合逻辑：通过连续书写实现”与”逻辑，如^\d[a-z]\d$要求严格按”数字+小写字母+数字”顺序匹配。
    • 匹配验证：当字符串为”1a2”时可匹配成功，若为”1a23”则因结束符限制匹配失败。

  • 多种规则的表达方式
    30:15

    • 或运算语法：使用|符号分隔不同规则，如\d[a-zA-Z]|[a-zA-Z]\d可匹配”数字+字母”或”字母+数字”两种模式。

  • 示例：或运算符（|）的使用

    • 实际应用：在字符串”g1”中，[a-zA-Z]\d规则可匹配成功；在字符串”1g”中，\d[a-zA-Z]规则可匹配成功。
    • 全局匹配：添加g标志可实现多次匹配，如同时匹配字符串中的”a1”和”1b”等组合。
4）规则的重复
  • 量词基本概念
    • 量词定义：表示前面规则出现次数的修饰符，如{3}表示前面规则精确出现3次。
  • 量词具体用法
    • 精确次数：[a-zA-Z]\d{3}匹配1个字母后跟3个数字（如”a123”）
    • 范围次数：{2,3}表示出现2-3次，{2,}表示至少2次
    • 特殊符号：
      • *等价于{0,}（零次或多次）
      • ?等价于{0,1}（零次或一次）
      • +等价于{1,}（一次或多次）
  • 分组量词应用
    • 分组匹配：用()将规则分组后加量词，如([a-zA-Z]\d){3}匹配3组”字母+数字”组合（如”a1b2c3”）
    • 作用范围：量词仅作用于紧邻的前一个规则或分组，需注意括号的使用范围。
5）应用案例

  • 例题:表单验证邮箱、库存、价格格式

    • 验证框架：使用validateHelper函数封装验证逻辑，通过回调函数返回错误信息
    • 邮箱验证：
      • 空值检查：if(!value) return ‘请输入邮箱’
      • 正则表达式：/^.+@[^\s.]+(.[^\s.]+){1,2}$/
        • 匹配规则：任意字符开头 + @ + 非空白字符域名 + 1-2级域名后缀
      • 测试方法：if(!mailReg.test(value)) return ‘邮箱格式错误’
    • 库存验证：
      • 空值检查：if(!value) return ‘请填写库存’
      • 正则表达式：/^[1-9]\d*$/
        • 匹配规则：非零数字开头 + 任意数字结尾
      • 测试技巧：可直接将正则表达式放入test方法
    • 价格验证：
      • 空值检查：if(!value) return ‘请输入价格’
      • 正则表达式：/^([1-9]\d*|0).\d{2}$/
        • 匹配规则：(非零整数或0) + 小数点 + 两位小数
      • 严格格式：必须包含两位小数，如19.00

  • 例题:文章段落包裹

    • 段落识别：
      • 特征分析：以多个空白字符开始，到下一个空白字符结束
      • 正则表达式：/\s+.+/g
        • 匹配规则：1+空白字符 + 任意字符
    • 处理流程：
      • 获取容器内容：container.innerHTML
      • 正则匹配段落：text.match(/\s+.+/g)
      • 包裹段落标签：s = ‘<p>‘ + s + ‘</p>‘
      • 移除空白字符：s.replace(/\s+/g,’’)
    • 实现技巧：
      • 使用全局匹配标志g处理多个段落
      • 通过replace方法同时完成匹配和替换
      • 注意处理最后一个段落可能缺少结束空白符的情况

二、替换空白字符

  • 方法：使用正则表达式s.replace()方法替换字符串中的所有空白字符
    • 操作步骤：先找到每个字符串中的空白字符，再将其替换为空字符或空格
    • 示例代码：s = s.replace(/\s+/g, ‘’) 将所有连续空白替换为空
    • 补充处理：替换后给字符串两端添加 <p>标签，形成 <p>处理后的文本 </p>结构
  • 应用场景：常用于清理HTML文本格式，使段落显示更规范
    • 处理后需要将新文本重新赋值给元素的innerHTML属性
    • 最终效果：使文章段落结构更清晰，每个段落都被 <p>标签包裹

三、内容总结

  • 技术要点：
    • 正则表达式是文本处理的强大工具，但使用频率不高容易遗忘
    • 实际开发中遇到复杂匹配需求时，建议查阅文档或参考示例
  • 学习建议：
    • 掌握基础正则语法即可，特殊用法使用时再查阅
    • 通过实际项目练习比死记硬背更有效
    • 重要技巧：处理HTML文本时注意保留原有标签结构
  • 扩展学习：
    • 更复杂的正则表达式应用可参考JavaScript高级教程
    • 常见使用场景：表单验证、文本搜索替换、数据清洗等
