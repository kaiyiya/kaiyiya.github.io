2）展开运算符

  • 例题:max函数使用

    • 基本语法：使用三个点…作为展开运算符，可以将数组或对象展开
    • 数组展开应用：
      • 示例：const arr = [3,6,4,7,2]
      • 传统调用：Math.max(3,6,4,7,2)需要手动展开数组元素
      • 展开运算符调用：Math.max(…arr)等价于上述传统调用
      • 原理：…arr会将数组元素逐个展开作为函数参数
    • 注意事项：
      • 不能直接传递数组给需要多个参数的函数
      • 展开运算符相当于”拆包”操作，将集合元素逐个展开

  • 例题:对象合并

      • 示例中o3结果为{a:3, b:2, c:4}
    • 展开过程：
      • …o1展开为a:1, b:2
      • …o2展开为a:3, c:4
      • 后展开的a:3会覆盖先前的a:1

      • 支持同时添加多个新元素和展开数组

3）解构

  • 例题:对象属性取出

    • 基本语法：使用大括号{}从对象中提取属性，语法为const {属性名} = 对象
    • 多属性提取：可以同时提取多个属性，如const {name, age} = user会创建两个常量name和age
    • 嵌套解构：对于嵌套对象，使用冒号表示层级关系，如const {addr: {city}} = user可提取深层属性
    • 变量类型：解构时可用const或let声明，使用const则变量不可修改
    • 传统方式对比：相比传统user.name写法，解构语法在多次使用时更简洁
    • 应用场景：当只需要使用对象的部分属性时特别有用
    • 组合解构：可以同时解构不同层级的属性，如const {name, addr: {city}} = user
    • 默认值：解构时可以设置默认值（虽然示例未展示，但MDN文档中有此特性）

  • 例题:数组解构

    • 基本语法：使用中括号[]从数组中提取值，如const [a,b,c,d] = arr
    • 跳过元素：使用逗号占位可跳过不需要的元素，如const [,a,b] = arr只取第二、三个元素
    • 间隔取值：可通过多个逗号实现间隔取值，如const [,a,,b] = arr取第二和第四个元素
    • 剩余元素：使用展开运算符…收集剩余元素，如const [a,b,…arr2] = arr
    • 变量交换：利用解构可轻松交换变量值：[b,a] = [a,b]
    • 实际应用：相比传统循环赋值，解构语法更简洁高效
    • 位置对应：解构变量按位置对应数组元素，与变量名无关
    • 灵活组合：可以自由组合各种解构方式满足不同需求

  • 例题:解构应用

    • 参数解构：可以在函数参数位置直接解构传入的对象或数组
    • 对象参数：如function method({a,b})会从传入对象中解构出a和b属性
    • 数组参数：如function method([a,b])会从传入数组中解构前两个元素
    • 箭头函数：箭头函数同样支持参数解构语法
    • 循环解构：在for…of循环中可以直接解构迭代项
    • 对象数组：处理对象数组时特别有用，如for(const {name,age} of users)
    • 简化代码：相比传统的item.property访问方式更简洁直观
    • 选择性使用：根据个人习惯选择是否使用，不强求必须使用新语法
    • 可读性优先：如果传统写法更清晰可读，可以继续使用传统方式
    • 渐进式采用：可以先从简单场景开始尝试使用解构语法
4）属性描述符

  • 例题：对象属性描述符

    • 基本概念：ES5提供的API，用于描述和操作对象属性的特性
    • 学习目的：主要为了理解Vue原理和应对面试，日常开发中较少直接使用
    • 描述符组成：
      • value：属性的值，如’name’属性的值为’monica’
      • configurable：是否可重新定义描述符，默认为true
      • enumerable：是否可被遍历（影响for-in循环），默认为true
      • writable：是否可修改属性值，默认为true

  • 例题：forIn遍历

    • 遍历机制：for-in循环能否遍历属性取决于enumerable的值
      • 示例：当enumerable为true时，可以输出name和age属性
      • 修改效果：将enumerable设为false后，该属性将不再出现在for-in循环中
    • 属性修改：writable控制属性值是否可修改
      • 默认情况：writable为true时可自由修改属性值
      • 限制情况：设为false后尝试修改会无效（不报错但值不变）

  • 例题：属性描述符操作

    • API方法：
      • Object.getOwnPropertyDescriptor(obj, prop)：获取属性描述符
      • Object.defineProperty(obj, prop, descriptor)：定义/修改属性描述符
    • 配置示例：
      • 将name属性的value改为’邓哥’
      • 设置writable为false使属性只读
      • 设置enumerable为false使属性不可遍历
      • 设置configurable为false后无法再次修改描述符
    • 特殊配置：getter和setter
      • get()：读取属性时调用，返回值作为属性值
      • set(val)：设置属性时调用，接收传入的值
      • 特点：使用getter/setter后属性行为变为方法调用
      • 存储技巧：通常需要配合内部变量存储实际值
    • 实现原理：
      • 定义内部变量存储实际值
      • get方法返回内部变量
      • set方法更新内部变量
    • Vue应用：Vue的响应式原理基于此特性实现数据绑定
5）对象API

  • 键值对操作：对象本质上是键值对的集合，键是属性名，值是属性值

  • 常用API：

    • Object.keys(obj)：获取对象所有属性名组成的数组
    • Object.values(obj)：获取对象所有属性值组成的数组
    • Object.entries(obj)：获取对象属性名和值组成的二维数组
    • Object.fromEntries(entries)：将二维数组转换回对象

  • 例题:对象API应用

    • 遍历技巧：
      • 使用for…of循环配合Object.entries()可以方便遍历对象
      • 结合解构赋值可清晰获取键值：for(const [key, value] of Object.entries(obj))
    • 实际应用：
      • 格式化输出：console.log(${key}:${value})
      • 数据转换：将对象转换为数组处理后再转回对象

  • 例题:Object.is用法

    • 对象冻结：
      • Object.freeze(obj)：冻结对象使其属性不可修改
      • 浅冻结：只冻结第一层属性，嵌套对象仍可修改
      • Object.isFrozen()：判断对象是否被冻结
    • 值比较：
      • Object.is()与===的区别：
        • NaN与NaN比较返回true
        • +0与-0比较返回false
      • 内部实现：ES6新增API(如Array.includes())都使用Object.is()进行比较
    • 开发建议：
      • 日常开发优先使用===
      • 特殊场景(如需要判断NaN)使用Object.is()
6）Set

  • 定义: Set是一种数据集合，用于保存一系列唯一的值，与数组不同，Set中的元素不会重复。

  • 特点:

    • 可以存储任何类型的唯一值，包括原始值和对象引用
    • 按照插入顺序迭代元素
    • 元素只会出现一次，保证唯一性

  • 相等性判断:

    • 在ECMAScript 2015规范前，+0和-0被视为不同值
    • NaN被视为相同的值，尽管
      NaN!==NaNNaN !== NaNNaN!==NaN
    • undefined也可以被存储在Set中

  • 基本用法

    • 创建Set:
      • 通过构造函数创建：const s = new Set()
      • 可以初始化：new Set([1,2,3])
    • 常用方法:
      • add(value): 添加元素，如s.add(1)
      • delete(value): 删除元素，如s.delete(3)
      • has(value): 检查元素是否存在
      • size: 获取元素数量
    • 注意事项:
      • Set没有下标概念，不能通过索引访问
      • 添加重复值时不会报错，但也不会生效
      • 可以通过values()方法获取迭代器

  • 例题：数组去重

    • 题目解析:
      • 需求：从一个包含重复元素的数组获取不包含重复项的新数组
      • 传统方法：需要编写循环遍历去重
      • 现代方法：利用Set特性简化操作
    • 解题思路:
      • 将数组转换为Set自动去重
      • 将Set转换回数组
    • 实现代码:
    • 关键技巧:
      • 利用展开运算符…将Set转换为数组
      • 整个过程可以简化为一行代码
    • 应用场景:
      • 数据清洗时去除重复项
      • 需要快速判断元素是否存在的场景
7）Map

  • 基本概念：Map是一种数据集合，用于保存一系列键值对(key-value pair)，其中键(key)是唯一的。
  • 与对象区别：
    • 键类型：对象的键只能是字符串或Symbol，而Map的键可以是任意类型（包括对象、函数等）
    • 键转换：对象会自动将数字键转换为字符串（如数组索引0会变成”0”），Map不会进行这种转换
    • 顺序保留：Map会记住键的原始插入顺序，而普通对象不保证顺序（ES6后对象也保留字符串键的插入顺序）
  • 键相等性判定
    • 比较算法：基于sameValueZero算法
      • NaN处理：NaN与NaN相等（尽管NaN !== NaN）
      • 零值处理：-0和+0被认为是相等的
      • 其他值：使用===运算符判断相等性
  • Map与Object对比
    • 键冲突：
      • Object可能有原型链上的键名冲突
      • Map默认不包含任何键，只包含显式插入的键
    • 大小获取：
      • Object需要手动计算键值对数量
      • Map可通过size属性直接获取
    • 迭代方式：
      • Map可直接迭代（iterable）
      • Object需要先获取键才能迭代
  • Map基本操作
    • 创建：const m = new Map()
    • 添加/修改：m.set(key, value)（键相同时会修改值）
    • 获取值：m.get(key)
    • 检查键：m.has(key)
    • 删除键：m.delete(key)
    • 清空Map：m.clear()
    • 获取大小：m.size

      • 键是唯一的，值可以重复

2. 应用案例
1）例题:对象书写属性与方法的应用

  • 属性速写：当对象属性名与变量名相同时，ES6允许简写形式，如{firstName}等价于{firstName: firstName}
  • 方法速写：对象方法可省略function关键字，直接写sayHello() {…}
  • 模板字符串应用：fullName属性使用模板字符串拼接${firstName}${lastName}
  • 解构赋值：函数返回对象后可通过const {fullName} = createUser()直接提取特定属性
2）例题:对象遍历的应用

  • 遍历属性名：
    • Object.keys(obj)获取属性名数组
    • 可用forEach遍历：Object.keys(obj).forEach(key => console.log(key))
  • 遍历属性值：
    • Object.values(obj)获取值数组
    • 同样支持forEach遍历
  • 遍历键值对：
    • Object.entries(obj)返回[key,value]数组
    • 可配合解构：Object.entries(obj).forEach(([k,v]) => console.log(k,v))
  • 对象复制：
    • 展开运算符实现浅拷贝：const newObj = {…obj}
    • 排除特定属性可先复制后删除，或用解构：const {a,…rest} = obj
3）例题:配置默认值的应用

  • 默认参数处理：使用options = options || {}处理未传参情况
  • 配置合并技巧：
    • 定义默认配置对象defaultOptions
    • 通过展开运算符实现覆盖：{…defaultOptions, …options}
    • 后展开的对象属性会覆盖前者同名属性
  • 经典模式：这种配置合并方式广泛应用于插件/库的选项配置
4）例题:对象属性Getter和Setter的应用

  • 属性改造流程：
    • 使用Object.entries()获取所有键值对
    • 遍历时通过Object.defineProperty重定义每个属性
    • 在getter中添加读取日志：正在读取${key}属性，值为${value}
    • 在setter中添加设置日志：正在设置${key}属性，新的值为${val}
  • 保持功能不变：
    • getter必须返回原值return value
    • setter必须更新值value = val
  • 复合操作处理：类似user.age += 10会先触发getter再触发setter
