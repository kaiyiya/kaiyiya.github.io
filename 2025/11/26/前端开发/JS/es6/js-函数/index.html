一、函数

1. 箭头函数
1）箭头函数

  • 基本语法：(参数列表) => {函数体}，例如求和函数可写为 const sum = (a, b) => { return a+b; }
  • 简化写法：
    • 当只有一个参数时，可省略小括号：参数 => {函数体}
    • 当函数体只有一条返回语句时，可省略大括号和return：(参数列表) => 返回语句
  • 箭头函数特点
    • 限制特性：
      • 不能使用new调用：箭头函数不能作为构造函数，使用new调用会报错
      • 没有原型：箭头函数没有prototype属性，sum.prototype返回undefined
      • 没有arguments：箭头函数内部arguments指向外层函数的arguments
      • 没有this：箭头函数内部的this指向外层函数的this
  • 适用场景
    • 最佳用途：特别适用于临时需要函数的位置，如回调函数
    • this绑定：箭头函数的this在定义时就确定了，不会随调用方式改变

  • 注意事项
    • 参数简写：单参数时可省略小括号，如const double = n => 2*n
    • 函数体简写：单条返回语句时可省略大括号和return，如(a,b) => a+b
    • 格式化问题：某些代码格式化工具会自动补全省略的小括号
2）例题：setTimeout箭头函数用法

  • this指向原理：箭头函数内部没有自己的this，使用的是外层函数的this。在示例中，setTimeout回调中的this指向user对象。
  • 与传统函数区别：普通函数有自己的this绑定，而箭头函数的this在定义时就确定了，不会随调用方式改变。
  • 实际应用：当需要保持this指向外层时特别有用，如事件处理、定时器等场景。
3）剩余参数
  • 基本概念
    • 替代arguments：ES6推荐使用剩余参数(…args)替代arguments对象来收集不定数量参数。
    • 语法特点：
      • 必须作为最后一个参数
      • 将剩余参数收集为数组
      • 可通过函数声明直观看出参数结构
  • 例题：剩余参数求和问题

      • 实际应用场景：如Math.max()等原生函数也采用类似参数处理方式
4）参数默认值

  • 基本用法

    • 定义规则：
      • 当参数未传递或传递undefined时使用默认值
      • 可以给任意参数设置默认值
      • 通常建议将有默认值的参数放在参数列表后面
    • 特殊注意：
      • 显式传递undefined会触发默认值
      • null不会触发默认值

      • 默认值在函数每次调用时重新计算
5）箭头函数特殊注意事项

  • 语法限制：
    • 不能用于立即执行函数表达式(IIFE)的特殊写法
    • 不能用作生成器函数(generator function)
  • 其他限制：
    • 不能使用new调用
    • 没有prototype属性
    • 没有arguments对象
    • 不能改变this绑定
6）类语法

  • 例题:用户类书写

    • 传统构造函数问题：
      • 任何函数都可以通过new调用或直接调用，无法从定义上区分构造函数和普通函数
      • 命名规范上使用首字母大写标识构造函数，但这不是强制约束
    • ES6类语法改进：
      • 使用class关键字明确定义类，与普通函数区分
      • 构造器使用特殊方法名constructor
      • 静态方法使用static关键字声明
      • 实例方法直接定义在类体中
    • 方法类型对比：
      • 静态方法：通过类本身调用（如User.isUser()），附着在构造函数上
      • 实例方法：通过实例对象调用（如u.sayHello()），定义在原型上
    • 代码组织优势：
      • 传统写法将相关功能分散定义（构造函数、静态方法、原型方法）
      • 类语法将所有功能集中定义，结构更清晰
    • 语法要点：
      • 类名遵循大驼峰命名规范
      • constructor方法处理初始化逻辑
      • 方法间不需要逗号分隔
      • 静态方法和实例方法采用统一的方法定义语法

  • 例题:动物类继承书写

    • 传统继承实现：
      • 使用Animal.call(this)在子类构造函数中调用父类构造函数
      • 需要手动设置原型链：Dog.prototype = Object.create(Animal.prototype)
      • 方法需要单独添加到原型上
    • ES6继承改进：
      • 使用extends关键字实现继承
      • 子类构造函数中必须调用super()来初始化父类
      • 方法直接定义在类体中
    • 继承优势：
      • 语法更简洁直观
      • 继承关系更明确
      • 减少了原型操作的复杂性
    • 注意事项：
      • 子类必须在constructor中先调用super()才能使用this
      • 静态方法也会被继承
      • 可以重写父类方法
    • 实际应用：
      • 父类定义通用属性和方法
      • 子类通过super调用父类构造函数
      • 子类可以添加特有属性和方法
      • 方法重写直接定义同名方法即可
2. 函数API
1）例题：函数调用方式应用

  • 数组slice方法：
    • 常规调用方式：arr.slice(start, end)，返回从start到end（不含end）的新数组
    • 示例：[1,2,3,4,5].slice(1,3) 返回 [2,3]
  • call/apply调用方式：
    • 通过Array.prototype.slice.call(arr,1,3)调用
    • apply与call区别：apply接收参数数组形式
  • bind绑定方式：
    • const slice = Array.prototype.slice.bind(arr)绑定this
    • 后续可直接slice(1,3)调用
    • 特点：返回新函数，永久绑定this
2）例题：求和函数书写

  • 箭头函数语法：
    • 完整形式：(参数列表) => {函数体}
    • 单参数简写：参数 => {函数体}
    • 单返回语句简写：(参数列表) => 表达式
  • 剩余参数应用：
    • 使用…args收集所有参数为数组
    • 示例：const sum = (…args) => args.reduce((a,b)=>a+b, 0)
  • 数组展开调用：
    • 使用展开运算符调用：sum(…[1,2,3])等效于sum(1,2,3)
    • 实现原理：将数组元素展开为独立参数
  • reduce方法对比：
    • 传统for循环：可读性较低但更直观
    • reduce方法：代码更简洁，需要适应
    • 建议：逐步过渡到函数式写法
  • 函数调用方式对比
    • call/apply/bind区别：
      • call：参数逐个传递，立即执行
      • apply：参数数组形式传递，立即执行
      • bind：返回绑定this的新函数，延迟执行
    • 实际应用场景：
      • 90%情况使用常规调用方式
      • 特殊场景（如React开发）可能需要bind
      • 理解原理比记忆语法更重要
  • ES6函数新特性
    • 常用程度排序：
      • 箭头函数（最高频使用）
      • 剩余参数（次常用）
      • 参数默认值（较少使用）
    • 开发建议：
      • 优先使用箭头函数替代传统函数表达式
      • 剩余参数处理不定参数更优雅
      • 参数默认值可增强函数健壮性
3）例题：学生数组处理
  • 使用箭头函数处理数组
    • 筛选电话号码以1结尾的学生：
      • 使用filter方法和箭头函数：const stus = students.filter(stu => stu.tel.endsWith(‘1’))
      • 简化技巧：箭头函数只有一条返回语句时可省略大括号和return关键字
      • 原理：endsWith方法判断字符串是否以指定字符结尾
    • 获取所有学生姓名数组：
      • 使用map方法：const names = students.map(stu => stu.name)
      • 映射本质：将学生对象数组转换为只包含姓名的字符串数组
  • 获取女生姓名和电话
    • 两步处理法：
      • 先用filter筛选女生：students.filter(stu => stu.sex === ‘女’)
      • 再用map转换格式：.map(stu => ({ name: stu.name, tel: stu.tel }))
    • 语法细节：
      • 箭头函数返回对象时必须加括号：() => ({…})
      • 原因：避免大括号被解析为函数体而非对象字面量
  • 生成HTML列表字符串
    • 实现步骤：
      • 使用map将每个学生转为 <li>标签：students.map(stu => ‘<li>${stu.name}</li>‘)
      • 用join拼接所有元素：.join(‘’)
    • 应用场景：
      • 将数据数组转换为可直接插入DOM的HTML字符串
      • 注意：模板字符串使用反引号`而非单引号
  • 综合技巧总结
    • 链式调用：合理组合filter、map、join等方法实现复杂数据处理
    • 箭头函数优势：
      • 使代码更简洁，特别适合作为回调函数
      • 自动绑定上下文，避免this指向问题
    • 练习建议：
      • 先尝试独立完成，再对照参考答案
      • 重点掌握数据转换的思维模式而非死记语法

6）例题：getDatas函数书写
  • 配置对象的使用方法
    • 配置对象概念：前端常见做法是将多个参数合并为一个配置对象传入函数，通过对象属性进行配置
    • 默认值处理：
      • 不传参数时默认值：page=1, limit=10, keyword=’’
      • 传递部分参数时：未传递的属性保持默认值
      • 示例：getDatas({page:2})会覆盖page默认值，其他保持默认
  • 传统实现方式
    • 参数处理：
      • 函数只接收一个config参数，默认值为空对象{}
      • 使用||运算符设置默认值：config.page || 1
      • 关键字处理使用三元运算符：keyword ? keyword : ‘空’
    • 输出格式：统一使用模板字符串输出结果
  • ES6解构赋值优化
    • 解构赋值优势：
      • 直接在参数位置解构并设置默认值
      • 语法更简洁：({page=1, limit=10, keyword=’’} = {})
      • 处理逻辑：未传递的属性自动使用解构默认值
    • 箭头函数简化：单条语句时可省略大括号
  • 两种实现方式对比
    • 传统方式：
      • 优点：逻辑清晰，易于理解
      • 缺点：代码量较多，需要手动处理每个属性
    • ES6方式：
      • 优点：代码简洁，语法糖使意图更明确
      • 缺点：需要理解解构赋值的默认值机制
