一、路由

1. 页面切换
  • 切换区域：在网站中切换页面实际上是切换中间主区域，其他区域（如左侧导航栏）保持不变
  • 传统方式：过去每个页面对应一个HTML文件（如首页index.html、文章页blog.html）
  • 现代方式：使用Vue开发时，通过JavaScript动态切换中间区域内容
2. 单页应用程序
  • 核心概念：整个应用只有一个HTML页面（index.html），通过JS动态构建内容
  • 实现方式：在空的div容器（如
    <divid=”app”></div><div id="app">``</div><**d**i**v**i**d**=**”**a**pp**”**><**/**d**i**v**>
    ）中通过JavaScript渲染不同组件
  • 优势：避免页面刷新带来的资源重新加载，提高用户体验和性能
3. 组件
  • 页面本质：在Vue中，每个页面实际上就是一个组件
  • 切换原理：路由切换的本质是不同组件的替换
  • 组件特点：页面组件通常较大，在概念上区别于普通功能组件
4. 工程结构
  • 目录规范：
    • 页面组件通常存放在views目录下（区别于components目录）
    • 可采用两种组织方式：
      • 直接创建.vue文件（如Home.vue）
      • 创建文件夹+index.vue（如Home/index.vue）
  • 示例结构：
    • views/
      • Home/（首页）
      • Blog/（博客）
      • About/（关于我）
      • Project/（项目）
      • Message/（留言板）
5. 路由插件的问题
1）组件的位置
  • 关键问题：
    • 如何根据URL路径选择对应组件
    • 如何确定组件渲染位置（如中间主区域）
  • 解决方案：
    • 使用Vue Router插件管理路由映射
    • 通过
      <router−view><router-view><**ro**u**t**er**−**v**i**e**w**>
      组件指定渲染位置
2）无刷新切换组件
  • 传统问题：页面刷新会导致所有资源重新加载
  • 现代方案：
    • 使用前端路由实现无刷新切换
    • 仅更新需要变化的部分（中间区域）
    • 通过history API或hash模式管理URL变化
3）路由插件使用
  • 安装方式：通过npm安装vue-router（
    npmivue−routernpm i vue-routernpmivue−router
    ）
  • 基本使用：
    • 在main.js中引入并使用插件
    • 创建路由实例并配置路由映射
    • 将路由实例挂载到Vue根实例
6. 路由插件基本使用
1）路由插件的使用

  • 插件安装方法：使用Vue构造函数的静态方法use()来安装插件，语法为Vue.use(插件对象)。例如安装vue-router插件时，需要先导入VueRouter对象。

  • 插件对象来源：插件对象是通过导入获得的，如import VueRouter from ‘vue-router’，然后将导入的VueRouter对象作为参数传递给use方法。

  • 配置路由

    • 路由实例创建：通过new VueRouter()创建路由实例，构造函数接收一个配置对象作为参数，该对象包含路由规则等配置信息。

    • Vue实例配置：创建好的路由实例需要配置到Vue实例中，在new Vue()时通过router选项进行配置，如new Vue({ router })。

    • 完整配置流程：

      • 使用Vue.use(VueRouter)安装路由插件
      • 创建路由实例const router = new VueRouter({…})
      • 将路由实例配置到Vue实例中new Vue({ router })

    • 常见错误：

      • 错误使用App.use()而不是Vue.use()会导致”use is not a function”错误
      • 仅安装插件而不配置路由实例会导致页面无任何效果

    • 路由匹配要求：

      • 必须明确配置路径与组件的对应关系
      • 需要指定组件渲染的位置（通常使用<router-view>组件）
      • 需要实现无刷新切换组件的功能

    • 组件渲染位置：在模板中使用<router-view>标签作为路由匹配组件的渲染出口。

2）路由模式

  • 路由配置与组件匹配

    • 配置结构：通过new VueRouter()创建路由实例，配置对象中包含routes数组定义路由规则

    • 路径匹配：每个路由规则是一个对象，包含path和component属性，如{ path: ‘/foo’, component: Foo }

    • 组件渲染：匹配到的组件会渲染到<RouterView />组件所在位置，该组件由vue-router全局注册

    • 插件安装：使用Vue.use(VueRouter)安装路由插件

    • 路由实例：创建的路由实例需要挂载到Vue根实例的router选项中

    • 模块化拆分：建议将路由配置拆分为单独模块，包含路由规则和路由实例的创建

  • 路由模式概述

    • 核心问题：路由模式决定两个关键问题：从哪里获取访问路径，以及如何改变访问路径
    • 模式类型：vue-router提供三种路由模式：hash、history和abstract
    • 无刷新原理：无论哪种模式，目标都是实现无刷新跳转，避免重新请求资源

  • hash模式

    • 路径获取：从浏览器地址栏的hash部分获取路径，如http://localhost:8081/#/blog获取/blog

    • 路径改变：通过修改location.hash改变路径，如location.hash = “#/blog”

    • 兼容性：兼容性最好，所有浏览器都支持hash变化不刷新页面

    • 监听机制：可以监听hashchange事件来响应路径变化

    • 特点：URL中始终包含#符号，路径变化只影响#后面的部分

    • 应用场景：适合需要兼容老旧浏览器的项目

  • history模式

    • 启用方式：在路由配置中设置mode: ‘history’

    • 路径获取：从location.pathname获取路径，如http://localhost:8081/blog获取/blog

    • 路径改变：使用HTML5 History API的history.pushState()方法

    • API使用：history.pushState(null, null, “/blog”)可以无刷新改变路径

    • 服务器配置：需要服务器端支持，确保所有路径都返回首页HTML

    • 兼容性：需要浏览器支持History API，现代浏览器都支持

    • 优势：URL更简洁美观，没有#符号

  • abstract模式

    • 适用环境：用于非浏览器环境，如移动端原生应用
    • 实现原理：路径保存在内存变量中，不依赖浏览器API
    • 特点：没有真实的URL变化，完全通过程序控制路径状态
    • 使用场景：在NativeScript、Weex等混合开发框架中使用
3）RouterLink组件

  • RouterLink的渲染结果

    • 组件本质：RouterLink是vue-router提供的全局组件，其渲染结果最终会转换为标准的<a>元素
    • 占位特性：与RouterView类似，都是vue-router提供的核心全局组件，RouterView负责显示匹配的组件内容，而RouterLink负责导航功能

  • RouterLink的链接地址属性

    • 属性区别：与传统a元素使用href不同，RouterLink使用to属性指定跳转目标地址
    • 使用示例：<RouterLink to="/blog">文章</RouterLink>会生成对应路径的超链接
    • 代码迁移：将项目中原本的a元素替换为RouterLink时，需要将href改为to属性

  • RouterLink在不同路由模式下的表现

    • hash模式：默认模式，生成形如<a href="#/blog">文章</a>的链接，兼容性最佳
    • history模式：生成标准路径链接<a href="/blog">文章</a>，但通过事件处理实现无刷新跳转
    • 自动适配：RouterLink会根据配置的路由模式自动生成对应格式的链接地址

  • RouterLink的点击事件处理

    • 事件拦截：RouterLink会为生成的a元素添加点击事件，并阻止默认的页面刷新行为
    • 内部机制：通过history API（pushState/replaceState）实现路径变更，确保单页应用的无刷新跳转
    • 实际效果：点击链接后URL变化但页面不刷新，仅更新RouterView区域的内容

  • RouterLink与a元素的对比

    • 模式解耦：使用RouterLink后，项目代码无需关心当前是hash还是history模式，组件会自动适配
    • 维护优势：切换路由模式时只需修改一处配置（mode选项），无需全局替换链接格式
    • 功能增强：相比原生a元素，额外提供激活状态管理、精确匹配等高级功能

  • RouterLink解除系统与模式间的耦合

    • 配置位置：在router/index.js中通过mode选项设置路由模式（’hash’或’history’）
    • 开发体验：开发者只需关注业务路径（如’/blog’），无需处理模式相关的路径格式差异
    • 切换示例：将mode从’history’改为’hash’后，所有RouterLink生成的链接自动变为hash形式

  • 激活状态管理

    • 匹配类名：
      • router-link-active：当前路径以导航路径开头时添加
      • router-link-exact-active：当前路径完全匹配导航路径时添加
    • 精确控制：可通过exact属性强制要求精确匹配才添加active类
    • 样式应用：这些类名可用于实现导航菜单的高亮效果，如<RouterLink to="/blog" exact>
4）激活状态

  • 路由导航激活机制

    • 匹配规则：
      • 模糊匹配：当前路径以导航路径开头时，添加router-link-active类名
      • 精确匹配：当前路径完全等于导航路径时，额外添加router-link-exact-active类名
    • 示例说明：
      • 访问/blog路径时：
        • /路径会获得router-link-active（模糊匹配）
        • /blog路径会同时获得router-link-active和router-link-exact-active

  • 匹配规则控制

    • exact属性：
      • 布尔属性，控制是否要求精确匹配
      • 添加后仅精确匹配时才会添加router-link-active
    • 实际应用：
      • 文章分类等需要模糊匹配的场景设为false
      • 首页、关于我等独立页面设为true

  • 类名自定义

    • 属性配置：
      • active-class：修改router-link-active的类名
      • exact-active-class：修改router-link-exact-active的类名
    • 实践建议：
      • 可通过active-class=”selected”简化类名
      • 不需要精确匹配样式时可置空exact-active-class

  • 实现原理

    • 路径获取：
      • 根据路由模式（hash/history）自动获取当前路径
      • 与导航路径进行逐项匹配
    • 匹配过程：
      • 先检查是否完全相等（精确匹配）
      • 再检查是否以导航路径开头（模糊匹配）
      • 最终根据匹配结果添加对应类名

  • 样式设置技巧

    • 样式优先级：
      • 精确匹配样式应覆盖模糊匹配样式
      • 使用!important确保样式优先级
    • 常见问题：
      • 多级路由需要特别注意匹配规则
      • 动态路由参数可能影响匹配结果

二、命名路由

1. 问题背景
  • 路径耦合问题：当项目开发后期需要修改路由路径时（如将”blog”改为”article”），所有使用该路径的超链接都需要同步修改，维护成本高。
  • 实际场景：产品经理要求将文章页面路由从/blog改为/article，导致所有相关链接失效。
2. 解决方案
  • 核心思想：通过给路由配置name属性解除与路径的耦合关系
  • 配置方法：
  • 跳转方式：使用对象形式传递路由名称而非直接写路径
3. 实现原理
  • 名称映射机制：路由系统会根据name属性自动查找对应的path进行跳转
  • 优势：
    • 路径可自由变更不影响业务逻辑
    • 保持业务含义的稳定性（名称代表业务含义）
    • 减少因路径变更导致的代码修改量
4. 注意事项
  • 命名规范：建议使用英文命名（如home/about/blog），避免使用中文
  • 动态绑定：必须使用v-bind（:）绑定to属性才能传递对象
  • 大小写：名称大小写需保持一致（建议统一大写或小写）

三、思维导图总结

1. Vue Router核心功能
  • 全局组件：
    • <router-view>：路由占位容器
    • <router-link>：导航组件
      • 属性：to/exact/active-class/exact-active-class
2. 路由配置要素
  • routes配置：定义路径与组件的映射关系
  • 路由模式：
    • hash模式（默认）
    • history模式
    • abstract模式（非浏览器环境）
3. 开发建议
  • 命名路由实践：建议所有路由都配置name属性
  • 路径变更流程：修改path后只需检查命名引用处
  • 组件解耦：避免在组件内直接使用路径字符串
