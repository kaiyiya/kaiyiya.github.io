一、图片水印

1. 上传时加水印

  • 方案一：静态水印

    • 流程：用户上传原始图片 → 服务器同时保存原始图片和加水印后的图片
    • 优点：用户请求时直接返回水印图片，响应速度快
    • 注意事项：必须保留原始图片，以便后续水印修改时重新生成
    • 缺点：水印变更时需要额外程序处理，需重新生成所有水印图片
  • 流程：仅保存原始图片，请求时实时生成水印
  • 核心优势：水印内容可随时修改，无需预处理
  • 性能影响：
    • 每次请求需读取图片到内存并修改数据
    • 高并发时内存压力显著增加（如100并发需复制100份数据）
  • 适用场景：水印需要频繁变更的情况
2. 图片处理技术实现
1）图片尺寸处理
  • 多尺寸存储：服务器可生成原始/大/中/小四种尺寸
  • 访问方式：通过不同URL地址返回对应尺寸图片
  • 实现价值：适配不同终端显示需求，优化加载性能
2）技术实现方案
  • 基础要求：需掌握图片二进制格式知识
  • 格式差异：JPG/PNG等不同格式存储方式不同
  • 推荐方案：使用专业图像处理库避免底层开发
3. JIMP库应用

  • 库特点：
    • 纯JavaScript实现，无原生依赖
    • GitHub星标10k+，社区活跃
    • 支持BMP/GIF/JPEG/PNG/TIFF等格式
  • 安装方式：npm install jimp
  • 核心功能：
    • 图像合成（blit）
    • 尺寸调整
    • 水印添加
    • 格式转换
4. 第三方库
1）jimp介绍

  • jimp使用方法

    • 模块导入：可通过require(‘jimp’)或ES6的import Jimp from ‘jimp’方式导入，TypeScript使用时需在tsconfig中设置allowSyntheticDefaultImports为true
    • 图片读取方式：
      • 支持回调模式：Jimp.read(‘图片路径’, (err, image) => {…})
      • 支持Promise模式：Jimp.read(‘图片路径’).then(image => {…}).catch(err => {…})
      • 支持多种输入源：本地图片路径、网络图片URL、Buffer数据或已有Jimp实例
    • 基本操作链：
      • .resize(width, height)：调整图片尺寸
      • .quality(百分比)：设置JPEG质量
      • .greyscale()：将图片转为灰度
      • .write(‘输出路径’)：保存处理后的图片
    • 图片处理方法
      • 尺寸调整：
        • .cover(w, h)：缩放图片覆盖指定尺寸
        • .resize(w, h)：直接调整尺寸，支持Jimp.AUTO自动计算
        • .scale(factor)：按比例缩放
        • .scaleToFit(w, h)：缩放至适应指定尺寸
      • 裁剪操作：
        • .autocrop()：自动裁剪同色边框
        • .crop(x, y, w, h)：裁剪指定区域
      • 合成操作：
        • .blit(src, x, y)：将另一张图片叠加到指定位置
        • .composite(src, x, y)：类似水印的合成效果
        • .mask(src, x, y)：使用另一张图片作为遮罩
      • 颜色处理：
        • .brightness(val)：调整亮度（-1到+1）
        • .contrast(val)：调整对比度（-1到+1）
        • .greyscale()：转为灰度
        • .invert()：颜色反转
      • Alpha通道：
        • .hasAlpha()：检测是否包含透明通道
        • .opacity(f)：设置透明度（0-1）
        • .background(hex)：设置默认背景色

  • 应用案例

    • 例题：哈士奇水印图片加水印

      • 准备工作

        • fade函数：作为opacity的替代方案，通过参数
          fff
          控制图片淡出效果，取值范围1-8，其中8表示完全不透明
        • 图片准备：
          • 原始图片命名为”orange.jpg”
          • 水印图片通过百度搜索获取（示例使用哈士奇图片），命名为”water.jpg”

      • 代码实现

        • 路径配置：
          • 使用path.resolve获取绝对路径
          • 定义三个关键路径：
            • waterPath：水印图片路径
            • originPath：原始图片路径
            • targetPath：处理后保存路径（示例为”new.jpg”）
        • 图片读取：
          • 使用Promise.all并行读取两张图片
          • jimp.read()返回Promise对象，解析后得到Jimp实例

      • 图片合成

        • composite方法：
          • 语法：origin.composite(water, x, y, options)
          • 参数说明：
            • x,y：水印位置坐标（示例使用10,10）
            • options：包含混合模式和透明度设置
        • 混合模式：
          • 默认使用Jimp.BLEND_SOURCE_OVER（源图像覆盖在目标图像上）
          • 其他可选模式包括相乘、屏幕、叠加等12种混合方式
        • 透明度控制：
          • opacitySource：源图像透明度（0-1），示例设为0.3
          • opacityDest：目标图像透明度（通常保持默认1）

      • 结果保存

        • 保存操作：
          • 使用origin.write(targetPath)保存处理后的图片
          • 注意await异步操作完成
        • 常见错误：
          • 大小写敏感：jimp应为小写
          • 路径错误：确保图片文件存在于指定路径

    • 图片加水印功能实现

      • 功能概述

        • 核心功能：通过Jimp库实现图片水印添加功能，包含水印位置计算、比例缩放等核心算法
        • 参数设计：
          • 水印图片(waterFile)
          • 原始图片(originFile)
          • 目标文件(targetFile)
          • 水印比例(proportion)
          • 边距比例(marginProportion)

      • 核心实现步骤

        • 基础函数框架

          • 函数定义：async function mark(waterFile, originFile, targetFile, proportion = 10, marginProportion = 0.05)
          • 参数说明：
            • proportion：水印与原始图片的宽度比例，默认10:1
            • marginProportion：水印距离边缘的比例，默认0.05（即5%）

        • 图片读取与缩放

          • 并行读取：使用Promise.all同时读取水印和原始图片

      • 水印位置计算

        • 边距计算：
          • 右边距：origin.bitmap.width * marginProportion
          • 底边距：origin.bitmap.height * marginProportion
        • 坐标计算：
          • x坐标：origin.bitmap.width - right - water.bitmap.width
          • y坐标：origin.bitmap.height - bottom - water.bitmap.height

      • 水印合成与输出

        • 合成参数：
          • mode: jimp.BLEND_SOURCE_OVER - 混合模式
          • opacitySource: 0.3 - 水印透明度
        • 输出保存：await origin.write(targetFile)

      • 应用场景

        • 上传时加水印：用户上传图片后，服务器同时保存原始图片和带水印版本
        • 动态水印：仅在请求图片时实时添加水印，不保存水印版本
        • 比例控制：通过proportion参数控制水印大小，适应不同尺寸的原始图片

      • 参数调优建议

        • 比例参数：
          • 默认proportion=10表示水印宽度是原图的1/10
          • 可调整为5使水印更大（1/5比例）
        • 边距参数：
          • 默认marginProportion=0.05（5%边距）
          • 可调整为0.01（1%边距）使水印更靠近边缘

    • 图片加水印并上传实现

      • 文件上传配置

        • 存储路径设置：
          • 原始图片保存到./public/origin目录
          • 水印图片保存到./public/upload目录
        • 文件名生成规则：
          • 采用”时间戳-6位随机字符.文件后缀”格式
          • 使用Date.now()获取时间戳
          • 通过Math.random().toString(36).slice(-6)生成随机字符串
        • 文件过滤：
          • 白名单限制为.jpg, .gif, .png格式
          • 文件大小限制为150KB（可通过移除限制解除）

      • 水印处理流程

        • 处理时机：在文件上传完成后立即执行
        • 文件路径管理：
          • 原始文件路径：req.file.path
          • 目标文件路径：通过path.resolve(__dirname, “../public/upload”, req.file.filename)构建
        • 异步处理：
          • 必须使用await等待水印处理完成
          • 水印函数mark()需要三个参数：水印图片路径、原始图片路径、输出路径

      • 水印图片配置

        • 水印文件位置：
          • 存放在./public/img/water.jpg
          • 通过path.resolve(__dirname, “../../public/img/water.jpg”)获取绝对路径
        • Git忽略设置：
          • 在.gitignore中添加public/origin目录
          • 防止原始图片被提交到版本库

      • 实现细节注意事项

        • 文件缓冲区：
          • 使用磁盘存储时没有buffer属性
          • 必须通过文件路径访问原始图片内容
        • 路径处理技巧：
          • 使用path.resolve处理跨平台路径问题
          • __dirname获取当前文件所在目录
          • 通过../返回上级目录

      • 完整上传接口实现

        • 接口响应：
          • 返回格式包含code, msg, data三个字段
          • data中返回处理后的图片访问路径
        • 错误处理：
          • 文件类型不符合时抛出Error(‘your ext name of${extname} is not support’)
          • 文件大小超限时自动拒绝（当启用限制时）

二、Jimp图像处理库功能详解

1. 图像合成与混合模式

  • 合成模式：Jimp提供多种图像混合模式，默认使用Jimp.BLEND_SOURCE_OVER
  • 常用模式：
    • Jimp.BLEND_MULTIPLY（正片叠底）
    • Jimp.BLEND_SCREEN（滤色）
    • Jimp.BLEND_OVERLAY（叠加）
    • Jimp.BLEND_DARKEN（变暗）
    • Jimp.BLEND_LIGHTEN（变亮）
  • 参数配置：可通过opacitySource和opacityDest分别控制源图像和目标图像的不透明度
2. 文字处理功能
  • 字体支持：使用BMFont格式(.fnt)支持多语言位图字体
  • 文字绘制：
    • 基础绘制：指定字体、位置和消息内容
    • 自动换行：通过maxWidth参数控制文本最大宽度
3. 图像特效处理

  • 常用特效：
    • image.posterize(n)：n级色调分离效果
    • image.sepia()：应用棕褐色调
    • image.pixelate(size)：像素化效果
    • image.displace(map, offset)：基于位移图扭曲图像
  • 操作建议：不可逆操作建议先克隆原图image.clone()
4. 图像缩放与裁剪
  • 缩放模式：
    • image.resize(w, h)：基础缩放
    • image.contain(w, h)：保持比例填充指定尺寸
    • image.cover(w, h)：保持比例覆盖指定尺寸
    • 支持Jimp.AUTO自动计算对应维度
  • 裁剪方法：
    • image.autocrop()：自动裁剪同色边框
    • image.crop(x, y, w, h)：指定区域裁剪
  • 实践技巧：上传时可生成多尺寸版本（如big.jpg、mid.jpg、mini.jpg）
5. 图像合成与变换
  • 合成操作：
    • image.blit()：图像拼接
    • image.composite()：图像叠加
    • image.mask()：使用另一图像作为蒙版
  • 几何变换：
    • image.flip(horz, vert)：水平/垂直翻转
    • image.rotate(deg)：旋转指定角度
  • 颜色调整：
    • image.brightness(val)：亮度调整（-1到+1）
    • image.contrast(val)：对比度调整
    • image.greyscale()：灰度化
    • image.invert()：颜色反转
6. 水印处理方案
  • 两种实现方式：
    • 静态水印：上传时生成带水印版本永久保存
    • 动态水印：请求时实时添加水印
  • 命名规范：
    • 原图：original.jpg
    • 大图：big.jpg
    • 中图：mid.jpg
    • 小图：mini.jpg
  • 开发建议：水印图宽度建议为原图的1/10比例
7. 文件上传实现

  • 安全控制：
    • 白名单校验文件扩展名（.jpg/.gif/.png）
  • 水印处理流程：
    • 接收上传文件
    • 指定水印图路径
    • 生成新文件路径
    • 调用水印添加函数
    • 返回处理结果

三、知识小结

知识点 核心内容 技术实现方案 优缺点对比 应用场景
图片水印技术 服务器端处理用户上传图片时添加版权标识 1. 上传时生成水印图片; 2. 动态请求时实时添加 方案1优势：请求响应快;方案2优势：水印可动态调整 用户生成内容(UGC)平台
图片处理库应用 使用Jimp库进行图像处理 - 读取/写入图片; - 尺寸调整/裁剪; - 图片合成(水印) 优势：支持多种输入源;局限：增加服务器内存负载 需批量处理图片的Web应用
水印实现细节 智能缩放与定位算法 - 按原始图比例缩放水印; - 动态计算右下角定位坐标 关键技术：composite()合成方法;参数控制：透明度/混合模式 需要精细化水印控制的场景
多尺寸图片生成 上传时自动生成不同尺寸版本 - 保留原始图; - 生成大/中/小缩略图 存储优化：后缀命名体系;性能权衡：预处理vs实时处理 响应式网站/移动端适配
二进制数据处理 图片作为二进制流处理 - 理解不同格式存储结构; - 使用Buffer对象操作 难点：JPG/PNG格式差异;解决方案：专用图像处理库 需要深度图像处理的专业系统
