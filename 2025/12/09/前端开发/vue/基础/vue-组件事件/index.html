一、分页组件

1. 嘁组件开发
1）开发方式
  • 静态优先原则：开发复杂组件时建议先完成静态页面和样式，再处理动态内容
  • 开发习惯差异：不同开发者有不同习惯（静态优先/数据优先），需根据个人偏好选择
  • 实现步骤：
    • 创建组件基本结构
    • 编写静态HTML模板
    • 添加CSS样式
    • 最后处理动态数据和交互逻辑
2）作用域样式对子组件根元素的影响

  • 命名规范：子组件根元素类名应采用”组件名-container”格式（如pager-container）

  • 设计原因：

    • 使父组件能清晰预测子组件根元素类名
    • 方便父组件通过scoped样式影响子组件根元素
    • 避免开发时频繁查看子组件源码确认类名

  • 实际影响：父组件中.a{…}的样式会同时作用于自身和子组件根元素的.a类

  • 规范实施：

    • Avatar组件根元素类名改为avatar-container
    • Icon组件添加icon-container类（保留必需的iconfont类）
    • Pager组件使用pager-container作为根元素类名
3）Avatar组件

  • 基本结构：

  • 属性定义：

    • url：必传字符串类型，指定头像图片地址
    • size：可选数字类型，默认150px，控制头像尺寸

  • 样式特点：采用容器类名规范，便于父组件样式控制

  • 配套修改：Icon组件同样遵循容器命名规范，同时保留功能必需的iconfont基础类

4）Pager组件
  • 基本结构：包含上一页、数字页码和下一页按钮
  • 特殊字符：
    • 使用<<表示首页
    • 使用<表示上一页
    • 使用>>表示尾页
    • 使用>表示下一页
  • 静态实现：初始阶段使用固定数字（如1-10）作为占位符，后续会改为动态生成
5）全局样式

  • 样式引入方式

    • 文件位置：在src/styles目录下创建global.less文件
    • 导入方式：在main.js中使用import “./styles/global.less”导入
    • 优势：相比第三方reset CSS更灵活，可根据项目需求定制

  • 全局样式内容

    • 基础样式：
      • text-decoration: none：去除下划线
      • color: inherit：继承父元素颜色
    • 交互效果：
      • 使用:hover伪类实现鼠标悬停变色效果
      • 颜色值推荐使用项目主色调（如#6b9eee）
    • 设计原则：
      • 全局样式应影响整个网站的一致性
      • 避免硬编码颜色值，建议使用CSS变量或主题色
6）应用案例

8）返回上级目录的写法

  • 问题背景: 当项目文件夹嵌套层次越来越深时，使用”../“返回上级目录的写法会变得繁琐不便

  • 解决方案: Vue CLI脚手架工具在webpack中配置了特殊字符表示src目录

  • 特殊字符表示src目录

    • 特殊字符: 使用ESC键下方的”~”符号（需按shift键）加上”@”表示src目录
    • 适用范围: 仅在样式文件(.less/.css等)中有效
    • 示例写法: @import “~@/styles/var.less”; 表示从src/styles目录引入var.less文件
    • 优势: 无论文件嵌套多深，都可以直接定位到src目录，避免使用大量”../“

  • 脚本中的路径写法

    • 区别: 在脚本文件(.js/.vue等)中不能使用”~@”写法
    • 正确写法: 直接使用”@”符号表示src目录
    • 示例: import “@/components/MyComponent.vue”
    • 注意: 样式文件和脚本文件使用不同的路径表示方法，这是Vue CLI的特殊配置
9）Pager组件样式实现

  • 布局方式: 使用flex布局使分页元素居中显示

  • 间距设置:

    • 上下边距20px
    • 左右边距6px（通过margin: 0 6px实现）

  • 颜色变量: 使用预定义的less变量@primary作为链接颜色

  • 禁用状态样式

    • 样式类名: .disabled
    • 视觉效果:
      • 颜色改为浅灰色(@lightWords)
      • 鼠标样式改为禁止符号(cursor: not-allowed)
    • 实现方式: 使用less嵌套语法&.disabled

  • 当前页样式

    • 样式类名: .active
    • 视觉效果:
      • 颜色改为深色文字(@words)
      • 字体加粗(font-weight: bold)
      • 鼠标样式改为文本选择(cursor: text)
    • 应用场景: 标识当前所在页码，与其他页码形成视觉区分
10）Pager组件属性与事件
  • 核心属性:
    • current: 当前页码，类型Number，默认值1
    • total: 总数据量，类型Number
    • limit: 每页显示数量，类型Number，默认值10
    • visibleNumber: 可见页码数，类型Number，默认值10
  • 关键事件:
    • pageChange: 页码变化事件，参数为新页码(Number类型)
11）组件属性

  • 组件文档与属性声明

    • 设计原则：组件开发前必须明确需要接收的属性，类似于函数参数
    • 文档要求：公司开发规范通常要求为每个组件编写文档，明确属性名称、类型、含义、是否必填和默认值
    • 属性设计要点：
      • 属性名应具有明确语义
      • 必须定义类型和默认值
      • 非必填属性需提供合理的默认值

  • Pager组件的属性详解

    • current：
      • 含义：当前页码
      • 类型：Number
      • 默认值：1（第一页）
      • 作用：决定哪个页码元素显示active样式
    • total：
      • 含义：总数据量
      • 类型：Number
      • 默认值：0
      • 注意：不是总页数，而是数据总条数
    • limit：
      • 含义：页容量（每页显示数据条数）
      • 类型：Number
      • 默认值：10
      • 计算关系：与total共同决定总页数
    • visibleNumber：
      • 含义：可见页码数（最多显示的数字页码数量）
      • 类型：Number
      • 默认值：10
      • 作用：控制分页器显示的数字页码数量

  • 属性的传递与默认值

    • 默认值设置：
      • 在props中使用default字段定义
      • 示例：current: { type: Number, default: 1 }
    • 数字属性传递：
      • 必须使用v-bind或:语法
      • 错误示例：<Pager current="2">（传递的是字符串）
      • 正确示例：<Pager :current="2">

  • 属性与界面渲染的关系

    • 首页按钮：
      • 动态class：{ disabled: current === 1 }
      • 逻辑：当前为第一页时禁用
    • 上一页按钮：
      • 同首页按钮逻辑
      • 使用相同条件判断
    • 尾页/下一页按钮：
      • 动态class：{ disabled: current === pageNumber }
      • 需要计算总页数：Math.ceil(total / limit)
    • 总页数计算：
      • 使用计算属性：pageNumber() { return Math.ceil(this.total / this.limit) }
      • 注意：必须向上取整，小数部分算作一页

  • 条件渲染

    • v-if与v-show

      • 使用场景：
        • 当总页数≤1时隐藏整个分页器
        • 示例：<div v-if="pageNumber > 1">…</div>
      • 实现方式：
        • v-if：完全移除DOM元素，适合不频繁切换的场景
        • v-show：通过CSS隐藏，适合频繁切换的场景
        • 分页器适合使用v-if，因为切换频率低

    • 动态class绑定

      • 对象语法：
        • 格式：:class=”{ className: booleanExpression }”
        • 示例：:class=”{ disabled: current === 1 }”
      • 多class控制：
        • 可以同时控制多个class
        • 示例：:class=”{ class1: cond1, class2: cond2 }”

  • 组件事件

    • 事件定义

      • pageChange事件：
        • 含义：页码变化时触发
        • 参数：新的页码（Number类型）
        • 使用场景：父组件监听页码变化并更新数据

    • 事件触发机制

      • 实现方式：
        • 子组件：this.$emit(‘pageChange’, newPage)
        • 父组件：<Pager @pageChange=”handlePageChange”/>
12）v-if和v-show

  • v-if指令介绍

    • 新增指令：v-if是Vue新增的指令，之前已学过三个指令（v-bind、v-on、v-for）
    • 作用原理：通过条件判断控制DOM元素的生成与销毁
    • 语法格式：v-if=”JS表达式”，表达式结果为布尔值，true时生成元素，false时销毁元素

  • v-if与v-else-if、v-else的使用

    • 分支结构：
      • v-if：主条件判断
      • v-else-if：次级条件判断（可多个）
      • v-else：默认情况（无需条件）
    • 使用要点：
      • 分支元素必须连续排列，中间不能插入其他元素
      • 每个分支对应不同的DOM元素
    • 示例场景：成绩评级系统
    • 数据驱动：通过修改score值（如30/60/90）自动切换显示结果

  • v-if控制元素显示隐藏

    • 单条件控制：
    • 响应式原理：通过修改data中的visible值（true/false）控制元素显示状态
    • 注意事项：
      • 变量名拼写需一致（如visible不能误写为visbile）
      • 控制的是数据而非直接操作DOM

  • v-if与v-show的区别

    • 核心区别：
      • v-if：条件为false时完全销毁DOM节点，不生成对应vnode
      • v-show：始终生成vnode和DOM节点，仅通过display:none控制显示
    • 性能影响：
      • v-if减少初始渲染量但导致虚拟DOM树不稳定
      • v-show保持DOM树稳定但无法减少渲染量
    • 使用场景：
      • 频繁切换（如选项卡）→ 优先使用v-show
      • 条件变化少（如初始化后很少改变）→ 优先使用v-if
    • 特殊注意：v-show不支持v-else语法，仅能单独使用

二、v-if与v-show的选择

  • 选择依据：根据条件变化的频率决定使用v-if还是v-show
  • 稳定条件：当条件（如总页数）不经常变化时使用v-if更好
  • 频繁变化：当条件（如当前页码）会频繁变动时使用v-show更合适
  • 性能差异：v-if会完全销毁/重建DOM元素，v-show只是切换CSS的display属性

三、数字数组生成

  • 实现思路：通过计算属性动态生成页码数字数组
  • 核心方法：使用v-for循环渲染数字数组
  • key值处理：必须为v-for添加唯一且稳定的key，可以使用数组下标
  • 样式绑定：通过动态class绑定active样式，判断条件是数字n等于当前页

四、计算属性应用

  • 最小页码计算：
    • 公式：min=current−⌊visibleNumber/2⌋min = current - \lfloor visibleNumber/2 \rfloormin=current−⌊visibleNumber/2**⌋**
    • 边界处理：最小值不能小于1
  • 最大页码计算：
    • 公式：max=min+visibleNumber−1max = min + visibleNumber - 1max**=min+visibleNumber**−1
    • 边界处理：最大值不能超过总页数
  • 数组生成：通过循环从min到max生成数字数组

五、组件通信规范

  • 单向数据流：子组件不能直接修改props接收的父组件数据
  • 问题原因：直接修改会影响父组件及其他可能使用该数据的组件
  • 正确做法：通过$emit触发事件通知父组件，由父组件决定是否修改
  • 类比说明：就像厨师不能擅自决定菜品去向，应该通知服务员处理

六、分页组件实现总结

  • 组件特点：通过props控制显示，用计算属性处理业务逻辑
  • 开发要点：
    • 区分稳定和频繁变化的条件选择v-if/v-show
    • 动态生成数字数组并处理边界情况
    • 遵守单向数据流原则处理交互

使用优势：一次开发后可以多处复用，只需通过props配置

七、组件事件

1. 事件概念与作用
  • 组件事件定义: 子组件发生特定行为时通知父组件的机制，如点击、定时触发等
  • 事件处理场景: 当子组件无法独立完成操作时（如分页组件需要修改父组件的列表数据）
  • 事件类比: 类似公司员工向上级提交请假申请/离职通知，需要附带相关数据（请假条/离职申请）
2. 事件抛出机制
  • 抛出方法: 使用Vue实例的$emit方法
  • 参数说明:
    • 第一个参数：自定义事件名称（如pageChange）
    • 第二个参数：事件参数（任意类型数据，如新页码）
  • 代码示例:
3. 事件注册与处理
  • 注册方式: 父组件通过@eventName监听子组件事件
  • 参数接收: 通过处理函数参数获取子组件传递的数据
  • 两种写法:
    • 直接写函数名：自动接收事件参数
    • 调用写法：需手动用$event传递参数
  • 处理逻辑: 父组件收到事件后更新自身数据（如currentPage），触发重新渲染
4. 事件参数验证
  • 边界检查:
    • 新页码不能小于1：newPage = Math.max(1, newPage)
    • 新页码不能超过最大页数：newPage = Math.min(totalPages, newPage)
  • 重复操作拦截: 当newPage === currentPage时直接返回不触发事件
5. 开发经验
  • 设计原则: 子组件只负责UI交互和事件通知，业务逻辑由父组件处理
  • 调试技巧: 在事件处理函数中添加console.log验证事件触发和数据传递
  • 学习曲线: 类比初次学习函数和回调的经历，需要时间适应组件化思维

八、课程总结

  • 核心要点: 掌握$emit的使用方法和事件通信流程
  • 实践建议: 完成分页组件开发，重点理解父子组件数据流向
  • 后续应用: 后续课程会大量使用组件事件，需牢固掌握基础概念
