一、导航栏制作 00:07

1. 导航栏组件创建
1）组件定义与引入 00:55
  • 组件封装必要性：需要单独对导航栏进行组件化封装，实现代码复用
  • 创建步骤：
    • 定义导航栏组件
    • 引入已定义的导航栏组件
2）组件结构搭建
  • 基础结构：
  • 样式处理：
    • 使用SCSS预处理器
    • 小程序需要特殊标签处理
    • 页面高度设置为100%
    • 采用flex布局方式
  • 图标处理：
    • 使用uni-icons图标插件
    • 安装地址：https://ext.dcloud.net.cn/plugin?id=28
3）兼容性处理
  • 状态栏高度问题：
    • 微信及APP需要进行状态栏高度占位处理
    • 参考方法：http://uniapp.dcloud.io/api/system/info?id=getsysteminfosync
  • 同名组件导入：
    • 使用easycom自动导入components/组件名/组件名.vue
    • 不需要使用import单独导入
4）项目配置准备
  • 云服务关联：
    • 右键关联云服务空间
    • 关联已定义的project store
  • 小程序配置：
    • 需在manifest.json中重新设置小程序APP ID
    • 否则打开小程序默认无显示
5）开发环境运行
  • H5运行：
    • 需要在界面下进行点击操作
    • 通过c运行命令启动
  • 小程序运行：
    • 通过微信开发者工具打开
    • 同时运行H5和小程序版本
  • 兼容性验证：
    • 需要同时运行多个环境验证导航栏显示效果
    • 不同平台可能存在样式布局差异
2. 创建导航栏组件 03:17
1）创建components目录 03:25
  • 目录规范：在uni-app项目中创建小写components目录用于存放组件
  • 文件管理：建议为每个组件创建同名目录，便于CSS等资源文件的单独管理
2）创建NavBar组件 04:09
  • 命名规范：组件名采用PascalCase命名法（如NavBar）
  • name属性作用：
    • 开发工具识别：方便在Vue DevTools中通过name快速搜索组件
    • 调试定位：当项目包含数十个组件时，能快速定位目标组件
    • 示例演示：将name改为”NavBar.page”后，在DevTools中可通过该名称搜索
3）引入组件 05:04
  • easycom机制：
    • 自动导入：符合components/组件名/组件名.vue路径规范的组件无需手动import
    • 局部引入：默认已实现组件按需加载，减少代码冗余
  • 传统引入方式：需要显式使用import和components注册
  • 结构搭建步骤：
    • 外层容器使用home-container类名
    • 直接使用<nav-bar>标签引入组件
    • 验证组件是否成功挂载（通过DevTools检查）
  • 调试技巧：
    • 在Vue DevTools中可查看组件层级关系
    • 通过name属性快速过滤目标组件
    • 支持实时修改组件数据并观察视图更新
3. 效果展示 07:15
  • 布局结构：固定定位的导航栏，包含搜索图标和输入框两个核心元素
  • 实现方式：使用uni-app框架开发，通过uni-icons插件实现图标功能
  • 功能特点：暂时不考虑搜索功能实现，先完成基础布局结构
4. 下载插件 08:11
1）插件获取流程
  • 获取途径：通过DCloud插件市场获取uni-icons图标插件
  • 搜索方法：在插件市场搜索”uni-icons”关键词
  • 版本选择：注意选择兼容当前HBuilderX版本的插件
2）插件安装步骤
  • 安装方式：
    • 在HBuilderX工具中选择”安装新插件”
    • 通过插件市场直接导入项目
  • 项目关联：需选择当前项目(project_level1)进行关联
  • 验证安装：在uni_modules目录下查看已安装的uni-icons插件
3）插件使用说明
  • 基本用法：直接使用<uni-icons>标签即可调用
  • 主要属性：
    • type：指定图标类型（如contact、search等）
    • size：控制图标大小（默认24px）
    • color：设置图标颜色
  • 事件处理：支持@click点击事件
  • 图标预览：可通过hello-uniapp示例工程查看全部可用图标类型
4）项目结构修改
  • 文件位置：在components/NavBar/NavBar.vue中添加图标组件
  • 实现步骤：
    • 在template中添加<uni-icons>标签
    • 设置合适的type属性值
    • 调整size和color属性匹配设计需求

注意事项：确保已正确导入uni_modules规范的插件

  • 注：所有时间戳均对应课程原始记录中的时间节点，图片标题根据PPT内容总结得出，知识点讲解完全基于教师字幕内容整理。
5. 编写导航栏结构 09:59
  • 外层容器：使用<view class="nav-bar">作为最外层容器，定义导航栏整体结构
  • 命名规范：类名采用nav-bar这种中划线连接方式，符合CSS命名最佳实践
1）站位设计 10:22
  • 固定定位处理：导航栏需要设置为固定定位(position: fixed)
  • 站位必要性：
    • 多端适配：需要支持小程序和APP等不同平台，各平台导航栏高度不一致
    • 动态计算：通过垫片区域动态计算高度值，避免在每个页面单独设置margin-top
  • 结构划分：
    • 展示区域：nav-bar-top包含实际展示内容
    • 垫片区域：空view用于撑开高度，保持布局稳定
2）内容区域实现
  • 内容容器：使用<view class="nav-bar-content">包裹实际内容
  • 图标组件：
    • 使用<uni-icons type="search">添加搜索图标
    • 设置图标颜色为#999的浅灰色
  • 文本区域：
    • 使用<view class="nav-bar-search-txt">包裹提示文本
    • 默认显示”输入文章标题进行搜索”
3）样式管理方案
  • 预处理器选择：使用SCSS编写样式(lang=”scss”)
  • 文件组织：
    • 推荐单独创建CSS目录管理样式
    • 组件样式文件命名为nav-bar.scss
  • 模块化优势：
    • 便于组件样式隔离
    • 方便后期维护和扩展
    • 支持样式变量和混入等高级特性
6. 编写NavBar组件样式
1）创建SCSS样式文件
  • 文件创建：在components/NavBar/css目录下新建NavBar.scss文件
  • 文件引入：在NavBar.vue中使用@import ./css/NavBar.scss引入样式文件
  • 语言设置：在style标签中设置lang=”scss”属性
2）编写navBar-top样式
  • 定位设置：
    • position: 使用fixed定位确保导航栏固定在顶部
    • top/left: 设置为0确保贴紧顶部和左侧
    • z-index: 设置为99保证导航栏在最上层
  • 尺寸设置：
    • width: 100%宽度占满屏幕
    • padding: 0rpx 30rpx 20rpx（使用rpx单位保证多端适配）
  • 背景设置：
    • background-color: 使用预定义的$base-color变量
    • box-sizing: border-box确保padding不影响总宽度
3）编写navBar-content样式
  • 布局设置：
    • flex布局：使用@include flex(flex-start)混入实现弹性布局
    • 高度：设置为60rpx
  • 视觉样式：
    • 背景色：纯白色(#FFFFFF)
    • 圆角：30rpx实现圆角效果
    • 内边距：左侧20rpx保证与图标的间距
  • 扩展性：
    • flex-grow: 1确保内容区域占满剩余空间
4）编写搜索框文本样式
  • 文本样式：
    • 字体大小：28rpx(相当于14px)
    • 颜色：使用$c-9变量(#999)设置浅灰色
  • 布局设置：
    • 宽度：100%占满父容器
    • 外边距：左侧20rpx与搜索图标保持距离
  • 注意事项：
    • 使用rpx单位确保多端适配
    • 颜色值使用变量便于统一管理
5）多端适配注意事项
  • 顶部间距问题：
    • 不同平台导航栏高度不同
    • 不能固定设置padding-top值
  • 解决方案：
    • 需要动态获取导航栏高度
    • 使用uni-app提供的API获取系统信息
  • 设计原则：
    • 保持样式在多平台的一致性
    • 使用相对单位(rpx)而非绝对单位(px)

二、动态添加样式 23:58

  • 动态样式需求：在导航栏组件中需要根据运行环境动态调整样式，特别是状态栏高度
  • 实现方法：通过uni-app提供的API获取系统信息，根据获取到的数据动态设置样式属性

三、获取系统信息 24:07

1. 获取窗口和系统信息 24:25
  • API使用：通过uni.getSystemInfoSync()同步获取系统信息，包含窗口尺寸、状态栏高度等关键数据
  • 文档参考：官方文档详细列出了各平台支持的属性，如状态栏高度(statusBarHeight)在字节跳动小程序不支持
2. 获取状态栏高度 24:35
  • 关键属性：系统信息对象中的statusBarHeight属性表示状态栏高度
  • 平台差异：微信小程序和原生APP可以获取该值，但H5端默认没有该属性
3. 初始化系统信息方法 24:49
  • 生命周期：在组件的created生命周期钩子中调用getSystemInfo方法
  • 方法定义：在methods中定义getSystemInfo方法，使用uni.getSystemInfoSync()获取系统信息
4. 打印系统信息 25:38
  • 调试方法：通过console.log打印获取到的系统信息对象
  • 查看结果：在控制台查看完整的系统信息，重点关注statusBarHeight属性值
5. 修复代码错误 25:56
  • 常见错误：data与methods之间缺少逗号会导致编译错误
  • 解决方法：确保组件选项之间使用逗号分隔，如data(){}与methods:{}之间
6. 状态栏高度获取与显示 26:12
  • 属性验证：在小程序控制台确认能正确获取到statusBarHeight值
  • 赋值使用：将获取到的状态栏高度赋值给组件数据属性，用于动态样式计算
7. 不同平台支持情况 26:26
  • 平台支持：
    • 微信小程序：支持获取状态栏高度
    • 原生APP：支持获取状态栏高度
    • H5端：不支持获取状态栏高度
    • 字节跳动小程序：不支持获取状态栏高度
8. H5端状态栏高度处理 26:42
  • 兼容方案：H5端没有状态栏高度时，可以设置默认值如10px
  • 条件判断：使用三元表达式处理：this.statusHeight = systemInfo.statusBarHeight ? systemInfo.statusBarHeight : 10 + ‘px’

四、默认值设定

1. 状态栏高度获取
  • 默认值设置：在data中定义statusHeight属性并设置默认值为40，这是为了后续进行rpx换算做准备。
  • 系统信息获取：通过uni.getSystemInfoSync()方法获取系统信息，其中包含状态栏高度(statusBarHeight)属性。
  • 平台差异：H5端获取的状态栏高度值为0，而小程序端可以正常获取到20（单位：px）。
2. 动态适配方案
  • 赋值方法：使用this.statusHeight = systemInfo.statusBarHeight将获取到的状态栏高度赋值给组件数据。
  • 调试技巧：通过console.log打印systemInfo对象，可以查看所有可用的系统信息属性。
  • 兼容处理：由于H5和小程序平台表现不同，需要使用动态计算的方式适配不同平台。
3. 实现要点
  • 生命周期：在created()钩子函数中调用获取系统信息的方法。
  • 数据定义：需要在data()中预先定义statusHeight属性作为响应式数据。
  • 动态计算：最终高度值应根据平台特性动态计算得出，而不是硬编码固定值。

五、使用垫片 28:11

1. 垫片的作用与实现
  • 本质：使用空view标签作为垫片，专门用于处理APP和小程序的默认导航栏问题
  • 实现方式：通过动态style设置高度值，实现响应式布局
  • 单位选择：使用rpx单位，因其能自动进行响应式转换
2. 动态样式设置
  • 语法格式：:style=”[height:statusHeight + ‘rpx’]”
  • 关键细节：
    • 必须添加引号（双引号）
    • 需要拼接单位’rpx’
    • 变量statusHeight来自组件data中定义的状态栏高度值
3. 常见问题处理
  • 常见错误：
    • 未加引号导致语法错误
    • 单位未正确拼接
  • 调试技巧：
    • 保存后刷新查看效果
    • 通过console.log验证statusHeight的值是否正确获取
4. 跨平台适配
  • 平台差异：
    • 小程序默认会推下内容
    • H5需要手动处理状态栏高度
  • 注意事项：
    • 获取的系统状态栏高度可能需要调整
    • 不同平台可能需要不同的处理逻辑

六、条件赋值与状态栏高度处理 30:03

  • 条件赋值逻辑：在获取系统状态栏高度时需先判断返回值是否存在，若H5等平台无高度值则使用默认值40px
  • 实现方法：通过uni.getSystemInfoSync()获取系统信息，使用三元运算符处理可能为空的情况
  • 默认值设置：在data中初始化statusHeight: 40作为备用值
1. 高度适配问题处理
  • H5适配方案：当获取不到状态栏高度时，采用20rpx的padding值保持布局对称
  • 小程序差异：小程序获取的statusBarHeight是20px值，需转换为rpx单位后数值会偏小
  • 临时解决方案：在小程序中可将获取的高度值乘以2进行初步适配
2. 胶囊按钮处理方案
  • 特殊处理需求：小程序右上角胶囊按钮会与导航栏内容产生重叠
  • API使用：通过uni.getMenuButtonBoundingClientRect()获取胶囊按钮布局信息
  • 关键参数：
    • width：胶囊按钮宽度（px）
    • height：胶囊按钮高度（px）
    • top：上边界坐标（px）
  • 布局计算：根据胶囊按钮的top值动态调整导航栏高度
3. 多平台适配要点
  • H5处理重点：确保默认高度和padding值使布局对称
  • 小程序处理重点：
    • 状态栏高度单位转换（px→rpx）
    • 胶囊按钮区域避让
    • 动态计算总高度
  • 通用方案：使用条件编译或运行时环境判断实现差异化处理

七、胶囊信息获取 32:13

1. API功能说明
  • 专用API: 使用getMenuButtonBoundingClientRect接口专门获取小程序右侧胶囊按钮的尺寸和位置信息
  • 获取内容: 可获取胶囊的宽度、高度以及距离屏幕上、下、左、右四个方向的距离值
  • 平台限制: 该API仅在小程序环境有效，APP和H5环境需要单独处理
2. 条件判断实现
  • 环境检测: 通过uni.getSystemInfo判断运行环境，使用条件语句if (process.env.UNI_PLATFORM === ‘mp-weixin’)识别微信小程序
  • 变量定义: 在小程序环境下定义menuButtonInfo变量存储胶囊信息
3. 布局适配方案
  • 状态栏高度: 将胶囊距离顶部的值top乘以2后赋值给statusBarHeight
  • 右边距调整: 动态设置marginRight值为胶囊宽度，避免内容遮挡
4. 垫片处理技巧
  • 遮挡问题: fixed定位会遮挡下方内容，需要通过动态样式解决
  • 高度计算: 垫片总高度 = 状态栏高度 + 搜索框高度(60rpx) + 内边距(20rpx)
  • 平台差异: 小程序循环从0开始计数，H5从1开始，需要注意显示差异
5. 搜索跳转实现
  • 事件绑定: 为搜索框内容区域绑定点击事件@click=”goSearchPage”
  • 跳转方法: 使用uni.navigateTo实现页面跳转
  • 页面创建: 需先在pages.json中配置search页面路径，并创建对应页面文件

八、知识小结

知识点 核心内容 易混淆点/难点 技术实现要点
导航栏组件创建 在components目录新建NV bar组件，使用easy component自动导入机制 同名组件导入方式与常规组件差异 新建组件时勾选”创建同名目录”
多端兼容性处理 动态获取statusBarHeight处理不同端状态栏高度差异 H5端无状态栏需设置默认值 uni.getSystemInfoSync()获取系统信息
胶囊按钮适配 微信小程序需额外获取menuButtonBoundingClientRect 像素单位转换(rpx与px比例) 胶囊宽度×2作为marginRight
定位与占位策略 固定定位导航栏+动态高度垫片防止内容遮挡 垫片高度=状态栏高度+内容区高度 :style=”{height: statusHeight + 80 + ‘rpx’}”
