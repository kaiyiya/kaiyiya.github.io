一、消除回调

1. ES7推出了两个关键字 async 和 await
1）async

  • 作用：ES7(ES2016)引入async关键字，用于更优雅地表达Promise异步操作
  • 语法要求：必须标注在函数前边，可以修饰各种函数形式：
    • 普通函数：async function fn(){}
    • 箭头函数：const fn = async () => {}
    • 函数表达式：const fn = async function(){}
    • 立即执行函数：(async function(){})()
  • 返回值特性：被async修饰的函数100%返回Promise对象
    • 无return语句时返回Promise { undefined }
    • 有return时返回值会成为Promise的完成值
    • 函数执行报错时返回rejected状态的Promise
2）await

  • async关键字的作用

    • 返回值转换：
      • 返回非Promise值：return 1 → Promise { 1 }
      • 返回Promise对象：保持原Promise状态
      • 抛出错误：throw new Error() → rejected状态的Promise
    • 语法糖本质：没有新增功能，只是让Promise使用更简洁
      • async function m(){return 123} 等价于 function m(){return Promise.resolve(123)}

  • await关键字的作用

    • 核心功能：等待Promise完成，获取完成值
    • 执行机制：
      • 暂停async函数执行，等待Promise完成
      • Promise完成后继续执行后续代码
      • 整个表达式返回Promise的完成值

  • await必须用于async函数中

    • 语法限制：
      • 只能在async函数内部使用
      • 顶级作用域直接使用会报SyntaxError
    • 原因：await需要暂停函数执行，普通函数不具备这种能力

  • await等待Promise完成

    • 典型用法：
    • 优势：消除回调嵌套，代码更线性易读
    • 等价转换：const n = await p 等价于 p.then(n => {…})

  • await也可以等待其他数据

    • 自动包装：非Promise值会被自动包装为已完成的Promise
      • await 1 → await Promise.resolve(1)
    • 使用建议：虽然支持但建议明确使用Promise.resolve()

  • 处理失败的Promise

    • 错误处理：
      • await遇到rejected Promise会抛出异常
      • 使用try-catch捕获处理错误
    • 对比传统方式：比.then().catch()链式调用更直观

二、知识小结

知识点 核心内容 考试重点/易混淆点 难度系数
Promise的作用 1.消除回调地狱（链式编程替代嵌套回调）; 2.统一异步处理方式（所有异步场景均可使用Promise） 回调未消除（Promise仍依赖回调，但优化了调用方式） ⭐⭐⭐
Async/Await语法 1.async标记函数：强制返回Promise; 2.await等待Promise完成：同步化异步代码; 3.错误处理：通过try-catch捕获异常 await必须用在async函数内（顶级作用域暂不支持） ⭐⭐
Promise与Async/Await对比 -Promise：需链式调用.then(); -Async/Await：无回调，代码更线性 本质均为语法糖（底层仍基于Promise） ⭐⭐
异步任务统一处理 1.封装异步操作（如delay函数）; 2.循环+Await实现顺序执行（如批量表白案例） 循环中await会阻塞后续迭代 ⭐⭐⭐⭐
实际应用场景 1.远程数据获取（如王者荣耀英雄数据API）; 2.定时任务控制（三次延迟输出案例） 错误处理优先级（失败需中断循环） ⭐⭐⭐
