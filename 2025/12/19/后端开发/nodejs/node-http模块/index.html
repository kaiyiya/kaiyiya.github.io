一、http模块

1. http.createServer

  • 基本语法: http.createServer([options][, requestListener])，返回一个新的http.Server实例
  • 参数说明:
    • options: 可选对象，包含三个配置项
      • IncomingMessage: 指定使用的IncomingMessage类，默认为http.IncomingMessage
      • ServerResponse: 指定使用的ServerResponse类，默认为http.ServerResponse
      • insecureHTTPParser: 布尔值，是否使用不安全的HTTP解析器（默认false，不建议使用）
    • requestListener: 自动添加到’request’事件的函数
1）request事件

  • 事件参数:
    • request: http.IncomingMessage对象，表示客户端请求
    • response: http.ServerResponse对象，用于向客户端发送响应
  • 对象关系总结:
    • 客户端视角:
      • 请求：ClientRequest对象（主动发出）
      • 响应：IncomingMessage对象（接收到的响应）
    • 服务器视角:
      • 请求：IncomingMessage对象（接收到的请求）
      • 响应：ServerResponse对象（主动发出的响应）
2）应用案例
  • 例题:监听服务器
    • 实现步骤:
      • 创建HTTP服务器并监听请求事件
      • 解析请求信息：
        • 使用url.parse()解析请求URL
        • 通过req.method获取请求方法
        • 通过req.headers获取请求头
        • 通过流式读取获取请求体
      • 启动服务器监听指定端口
    • 关键代码:
    • 注意事项:
      • 请求体需要通过流式读取，因为可能包含大量数据（如文件上传）
      • 使用url.parse()可以方便地解析URL中的路径、查询参数等信息
      • 服务器需要显式调用listen()方法开始监听端口

二、服务器响应

1. 请求处理函数

  • 函数封装：将请求处理逻辑封装为handleReq函数，包含请求路径、方法、头部和请求体的完整获取
  • 数据读取：通过req.on(“data”)和req.on(“end”)事件流式读取请求体内容
  • 模块化处理：将处理函数独立出来便于后续按需调用特定部分
2. 服务器监听端口

  • 端口绑定：使用server.listen(9527)绑定服务端口
  • 监听事件：通过server.on(“listening”)回调确认服务启动成功
  • 请求处理：在http.createServer回调中调用处理函数handleReq(req)
3. 服务器响应对象

  • 可写流特性：res对象是可写流，通过写入操作实现响应
  • 响应组成：包含响应行、响应头和响应体三部分
  • 自动构建：相比net模块自动处理HTTP协议格式，无需手动构建消息结构
4. 设置响应头和消息体

  • 头设置方法：使用res.setHeader(“a”, “1”)设置自定义响应头
  • 消息体写入：通过res.write(“你好！”)写入响应体内容
  • 结束标记：必须调用res.end()表示响应结束
  • 默认头部：系统会自动添加Date、Connection等默认响应头
5. 响应结果查看

  • 状态码验证：默认返回200状态码
  • 头信息验证：自定义头a:1和b:2成功出现在响应中
  • 消息体验证：写入的”你好！”正确显示在响应体
6. 移除默认响应头

  • 头移除方法：使用res.removeHeader(name)移除特定响应头
  • 头查询方法：通过res.getHeader(name)查看已设置的响应头
  • 头管理：可以获取所有头名称getHeaderNames()和头对象getHeaders()
7. 设置响应状态码

  • 默认状态码：未设置时默认为200
  • 设置方法：直接赋值res.statusCode = 404修改状态码
  • 状态消息：会自动匹配标准状态消息（如404对应”Not Found”）
  • 效果验证：在Postman中可看到状态码变为404

三、静态资源服务器

1. 基本概念
  • 定义: 静态资源服务器是指将特定目录下的文件（如HTML、CSS、JS、图片等）直接提供给客户端的服务器
  • 特点:
    • Node.js不会执行这些文件，仅作为文本文件读取
    • 浏览器负责解析和执行这些资源
    • 常见的静态资源包括：index.html、a.js、style.css、images等
2. 实现原理
  • 核心机制:
    • 根据请求路径映射到public目录下的对应文件
    • 例如请求/index.html则返回public/index.html文件内容
    • 请求/css/style.css则返回public/css/style.css文件内容
  • 与传统开发区别:
    • 不同于Live Server等插件自动创建的服务器
    • 需要手动实现路径映射逻辑
3. 实现步骤
1）基础服务器搭建
  • 关键点:
    • 使用Node.js原生http模块
    • 创建服务器并指定处理函数
    • 监听6000端口（可自定义）
2）请求路径处理
  • 路径转换:
    • 使用URL模块解析请求路径
    • 去除路径开头的斜杠（避免根路径问题）
    • 拼接public目录形成完整文件路径
3）文件存在性检查
  • 异常处理:
    • 使用fs.stat检查文件状态
    • 捕获异常返回null表示文件不存在
    • 区分目录和普通文件
4）默认文件处理
  • 特殊规则:
    • 请求目录时自动查找index.html
    • 请求根路径/时返回public/index.html
    • 示例：请求/css/会尝试返回public/css/index.html
5）响应处理
  • 响应逻辑:
    • 文件不存在返回404状态码
    • 文件存在直接返回文件内容
    • 使用fs.readFile读取整个文件（实际生产建议使用流）
4. 注意事项
  • 端口问题: 避免使用已被占用的端口（如6000可能被重定向）
  • 性能考虑: 大文件应使用流式读取
  • 安全考虑: 需要防止路径遍历攻击
  • MIME类型: 实际应用应设置正确的Content-Type

四、课程总结

  • 核心收获: 掌握了静态资源服务器的实现原理和编码方法
  • 关键技能:
    • http模块创建服务器
    • 路径解析和文件映射
    • 异步文件操作
    • 错误处理和状态码设置
  • 练习建议: 熟练掌握http模块的请求处理和响应方法

五、知识小结

知识点 核心内容 重点/易混淆点 难度系数
HTTP模块基础 建立在TCP/IP之上的应用层协议 与TCP模块的区别：HTTP自动管理socket连接和消息格式 ⭐⭐
客户端请求 使用http.request()方法发送请求 非Ajax请求：Node.js的HTTP请求是纯粹的HTTP请求，不存在跨域问题 ⭐⭐⭐
请求体处理 GET请求无消息体，POST需手动写入 关键操作：必须调用request.end()表示请求结束 ⭐⭐⭐⭐
响应处理 响应头通过response.headers获取 流式读取：响应体需通过流的方式分块读取，避免内存问题 ⭐⭐⭐⭐
服务器搭建 http.createServer()创建服务器 请求/响应对象：区分IncomingMessage(请求)和ServerResponse(响应) ⭐⭐⭐
静态资源服务器 根据URL路径映射本地文件 路径处理：需处理文件不存在、目录等情况，默认返回index.html ⭐⭐⭐⭐
状态码设置 通过response.statusCode设置 常见错误：未正确结束响应导致客户端一直等待 ⭐⭐
消息头操作 setHeader()/getHeader()方法 性能注意：避免在响应头设置过多不必要信息 ⭐⭐
流式处理 请求体和响应体都采用流式处理 核心优势：适合处理大文件，避免内存溢出 ⭐⭐⭐⭐
错误处理 文件不存在返回404状态码 用户体验：应提供友好的错误信息页面 ⭐⭐⭐
