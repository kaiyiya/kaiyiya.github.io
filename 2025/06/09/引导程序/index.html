初始引导扇区程序

这段汇编代码是一个简单的引导扇区程序（Boot Sector），用于计算机启动时的初始引导。采用16位实模式，符合x86架构的引导要求。以下是逐部分解释：

  1. #include "boot.h"
    包含头文件boot.h，通常用于定义常量或宏（此处未展示具体内容）。
  2. .code16
    指示汇编器生成16位代码，这是计算机启动时BIOS执行的实模式要求。
  3. .text 与入口点
    _start: 是程序入口点，BIOS将引导扇区加载到内存0x7C00后从这里开始执行。
  4. 初始化段寄存器
    • 将AX寄存器设为0，并复制到所有段寄存器（DS, SS, ES, FS, GS）。
    • 目的：使数据段、堆栈段等均从物理地址0开始（实模式下段地址×16 + 偏移量）。
      注意：此时CS（代码段）通常由BIOS设为0x07C0，对应物理地址0x7C00（引导扇区位置）。
  5. 设置栈指针
    mov $_start, %esp
    • 将栈顶指针ESP设为_start的地址（即0x7C00）。
      潜在问题：栈向下增长，若压栈可能覆盖0x7C00以下的引导代码。通常建议将栈设在其他安全区域（如0x7C00之前）。
  6. 无限循环
    jmp .
    跳转到当前指令（.表示当前地址），形成死循环。实际引导程序会在此处加载操作系统，但本代码仅作演示。
  7. 引导结束标志
    • 自定义段boot_end（属性ax表示可分配、可执行）。
    • 定义两字节0x55, 0xaa：这是引导扇区必需的结束标志（Magic Number），位于第510和511字节，供BIOS验证扇区有效性。

关键注意事项：

  • 引导扇区大小：整个程序需严格占用512字节，末尾两字节必须是0x55 0xaa。
  • 实际作用：此代码仅初始化环境并挂起，无实际引导功能。完整引导程序会加载操作系统的后续部分。
  • 内存布局：
    • 代码从0x7C00（_start）开始执行。
    • 栈指针初始指向0x7C00（存在覆盖风险）。
    • 结束标志0x55 0xaa由链接脚本确保位于第510字节处。

这段汇编代码实现了一个磁盘读取功能，主要用于从硬盘加载数据（如操作系统加载器）。以下是逐行解释：

磁盘读取功能

  1. 目标地址：0x8000是经典引导加载位置，BIOS将引导扇区加载到0x7C00后，第二阶段加载器通常放在0x8000
  2. BIOS磁盘参数：
    • AH=0x02：磁盘读取功能
    • AL=64：读取64个扇区（每扇区512字节，共32KB）
    • CH=0：柱面号0（磁道）
    • CL=2：起始扇区号2（扇区号从1开始）
    • DH=0：磁头号0
    • DL=0x80：第一块硬盘
  3. 错误处理：
    • jc read_loader：若操作失败（CF=1），跳回标签重试
    • 失败原因可能包括：磁盘错误、超时、无效参数等
  4. 物理位置计算：
    • 读取位置：柱面0/磁头0/扇区2开始
    • 这是典型的MBR分区方案中，主引导记录后的第一个可用扇区

代码详细解析

1. Boot模块 - start.S (汇编)

  • 核心作用：作为Boot模块的入口点（BIOS加载到0x7C00后执行的第一条指令）
  • 关键跳转：直接跳转到boot_entry函数（在boot.c中实现）
  • 执行位置：物理地址0x7C00（BIOS加载引导扇区的标准位置）

2. Boot模块 - boot.c (C语言)

  • 核心作用：跳转到Loader模块
  • 关键技术：
    • __asm__(".code16gcc")：让GCC生成16位实模式兼容的代码（保持代码段偏移量正确）
    • 函数指针跳转：(void (*)(void))将0x8000强制转换为函数指针类型
    • 函数调用：()执行该地址的代码（相当于汇编的call 0x8000）
  • 内存布局：
    • Boot模块在0x7C00（占用512字节）
    • Loader模块将被加载到0x8000（向上预留了约1KB空间）

3. Loader模块 - start.S (汇编)

  • 核心作用：作为Loader模块的入口点（位于0x8000）
  • 关键跳转：跳转到loader_entry函数（在loader_16.c中实现）
  • 内存位置：物理地址0x8000（由Boot模块加载）

4. Loader模块 - loader_16.c (C语言)

  • 当前作用：作为占位符（实际应包含加载操作系统的代码）
  • 预期功能：
    • 初始化保护模式
    • 设置GDT/IDT
    • 加载内核到内存
    • 跳转到内核入口

    • Loader模块入口_start位于二进制文件起始位置

    • 所有文件通过.code16/.code16gcc保持16位实模式
    • 确保函数调用使用正确的调用约定

典型执行流程

  1. BIOS加载Boot模块到0x7C00
  2. 执行start.S → 跳转到boot_entry()
  3. boot_entry() → 跳转到0x8000（Loader入口）
  4. Loader的start.S → 跳转到loader_entry()
  5. loader_entry()执行实际引导操作

关键点：这种设计实现了从汇编到C语言、从Boot到Loader的无缝跳转，为后续操作系统加载奠定了基础。实际项目中，loader_entry()应包含切换到保护模式、加载内核等关键操作。
